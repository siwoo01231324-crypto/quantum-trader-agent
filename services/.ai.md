# services/

## 목적
LLM·외부 에이전트 연동 레이어. 트레이딩 핵심 로직(`src/`)과 분리해 보조 기능만 여기에 둔다.

## 구조
| 디렉토리 | 역할 | 관련 이슈 |
|----------|------|-----------|
| `obsidian_mcp/` | Obsidian 볼트 stdio MCP 서버 — 7개 도구로 LLM 에 볼트 노출 | #51 |
| `doc_agent/` | 이벤트 기반 초안 노트 자동 생성 (백테스트·인시던트·포스트모템) | #53 |

## 경계
- `src/` 의 실행·리스크·체결 로직을 **직접 호출하지 않는다**. 필요하면 이벤트 큐·파일 기반 인터페이스 경유
- LLM 호출은 선택(`--with-llm` 플래그), 기본 동작은 템플릿 기반
- 쓰기 도구는 기본 dry-run, 사람이 승인 후에만 실쓰기
- 자동 커밋·푸시 금지

## 새 에이전트 추가 체크리스트
새로 에이전트를 만들 때마다 아래 한 줄(볼트 연결)을 빼먹지 말 것. 이 프로젝트 에이전트의 기본값은 **온톨로지 볼트를 알고 있는 상태**다.

1. **볼트 연결** — 둘 중 하나
   - in-process: `from services.obsidian_mcp import tools` → `tools.read_note / list_notes / search / sparql / graph_neighbors` 직접 호출 (`doc_agent` 가 이 방식)
   - 외부 프로세스 / 외부 LLM: `python -m services.obsidian_mcp.server` 를 stdio MCP 로 등록 (`docs/onboarding/mcp-setup.md`)
2. **쓰기 경로**: 기본 dry-run → 사람이 결과 검토 후 `--write` 또는 `OBSIDIAN_MCP_ALLOW_WRITE=1`. 신규 노트는 `.draft.md` 로만 생성
3. **스키마 준수**: 노트 생성·수정 시 `docs/schemas/note-schemas.md` 의 타입 중 하나로 프론트매터 구성. 위반하면 `scripts/check_invariants.py --strict` 가 CI 에서 차단
4. **`.ai.md` 등록**: `services/<agent>/.ai.md` 에 목적·입출력·볼트 사용 범위 기술. 볼트를 쓰지 **않는** 에이전트면 그 사유를 명시
5. **테스트**: `tests/fixtures/obsidian_vault/` 픽스처 재사용 — 실제 `docs/` 를 건드리지 말 것
