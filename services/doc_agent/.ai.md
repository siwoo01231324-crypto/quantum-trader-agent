# services/doc_agent/

## 목적
이벤트(백테스트 완료·인시던트 감지·포스트모템 요청) 발생 시 프론트매터 스키마(`docs/schemas/note-schemas.md`)를 준수하는 **초안 노트(`.draft.md`)** 를 자동 생성한다. MCP(#51) 미완료 상황에서도 동작하도록 파일시스템에 직접 쓴다.

## 구조
| 파일 | 역할 |
|------|------|
| `__init__.py` | 공개 API 재노출 (`generate_*_draft`) |
| `generators.py` | 3종 초안 생성기 구현 (`backtest`/`incident`/`postmortem`) |
| `prompts/backtest.md` | 백테스트 요약 LLM 프롬프트 |
| `prompts/incident.md` | 인시던트 root cause 초안 프롬프트 |
| `prompts/postmortem.md` | 포스트모템 timeline 재구성 프롬프트 |
| `cli.py` | `python -m services.doc_agent.cli {backtest|incident|postmortem}` 엔트리 |
| `llm.py` | anthropic SDK stub (`--with-llm` 플래그 시 호출) |
| `audit.py` | 실행 로그를 `docs/work/agent-runs/` 로 기록 |

## 인터페이스
- `generate_backtest_draft(bt_result_json: dict) -> Path`
  - 출력: `docs/work/done/backtests/bt-{YYYY-MM-DD}-{strategy}.draft.md`
- `generate_incident_draft(event: dict) -> Path`
  - 출력: `docs/work/incidents/inc-{YYYY-MM-DD}-{slug}.draft.md`
- `generate_postmortem_draft(incident_id: str) -> Path`
  - 출력: `docs/work/incidents/pm-{YYYY-MM-DD}.draft.md`
  - 해당 incident 노트를 읽어 백링크를 자동 수집

## 규칙
- 모든 출력 파일은 `.draft.md` 확장자 (`check_invariants.py` 가 스키마 검증 제외)
- 프론트매터 `status: draft` 명시 (해당 타입에 status 필드가 있는 경우)
- LLM 호출은 기본 비활성. `--with-llm` 플래그로만 활성화, SDK 부재 시 템플릿 폴백
- 실행할 때마다 `docs/work/agent-runs/{ts}-{generator}.log` 감사 로그 기록
- 서비스 내부에서 `src/` 로직에 의존하지 않는다 (초안 생성 전용 유틸)
