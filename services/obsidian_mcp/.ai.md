# services/obsidian_mcp — Obsidian 볼트 MCP 서버

## 목적
`docs/` Obsidian 볼트를 **MCP stdio 서버**로 노출해서, Claude Code / 외부 LLM 이
파일시스템을 직접 만지지 않고도 "도구" 로 볼트를 읽고/쓰고/쿼리할 수 있게 한다.

- 전략·시그널·리스크 규칙 등 프론트매터 스키마(`docs/schemas/note-schemas.md`)
  기반 CRUD
- `docs/ontology/trading.ttl` + `instances.ttl` 에 대한 SPARQL 쿼리
- 백링크·아웃링크 기반 그래프 탐색

## 구조
```
services/obsidian_mcp/
├─ __init__.py        # 패키지 엔트리 (tools 재노출)
├─ .ai.md             # 이 문서
├─ server.py          # MCP stdio 서버 (python -m services.obsidian_mcp.server)
└─ tools.py           # tool 구현체 7종 (순수 함수, 서버에서 래핑)
```

## 노출 도구 (7)
| tool | 설명 | 쓰기 |
|------|------|------|
| `read_note(id)` | 프론트매터(dict) + body(str) | - |
| `list_notes(type?, tag?, path_prefix?)` | id + 요약 리스트 | - |
| `search(query)` | 풀텍스트·태그·위키링크 매칭 | - |
| `write_note(id, frontmatter, body, create_if_missing)` | 노트 쓰기 | **dry-run 기본** |
| `append_section(id, heading, content)` | 섹션 추가 | **dry-run 기본** |
| `sparql(query)` | GraphDB 원격 또는 로컬 rdflib 으로 `trading.ttl + instances.ttl` 쿼리 (SELECT/ASK/DESCRIBE/CONSTRUCT 만 허용) | - |
| `graph_neighbors(id, depth=1)` | 위키링크 기반 이웃 노드 | - |

## 쓰기 모드
- 기본: **dry-run** — 변경 내용을 반환만 하고 파일을 쓰지 않는다
- 실쓰기 활성화 조건 (둘 중 하나):
  - 환경변수 `OBSIDIAN_MCP_ALLOW_WRITE=1`
  - CLI 플래그 `python -m services.obsidian_mcp.server --write`
  - 또는 `docs/.obsidian/mcp-config.json` 의 `write_mode: "enabled"`
- 허용 경로는 `mcp-config.json` 의 `allowed_paths` 로 화이트리스트

## 실행
```bash
# stdio 서버 (Claude Code 가 자식 프로세스로 띄우는 전형적 방식)
python -m services.obsidian_mcp.server

# 실쓰기 허용
OBSIDIAN_MCP_ALLOW_WRITE=1 python -m services.obsidian_mcp.server
python -m services.obsidian_mcp.server --write
```

Claude Code 등록: `docs/onboarding/mcp-setup.md` 참조.

## SPARQL 엔드포인트 우선순위 (#55)
1. `ctx.sparql_endpoint` (`mcp-config.json` 의 `sparql_endpoint`) — 프로그램적 최우선
2. `QTA_SPARQL_ENDPOINT` 환경변수
3. 로컬 rdflib (`trading.ttl + instances.ttl` 파싱)

원격 호출 시 **3계층 방어**: (i) 엔드포인트는 `/repositories/{repo}` (read) 만 — `/statements` 절대 호출 안 함 (ii) 로컬 경로는 `Graph.query()` 고정, `Graph.update()` 금지 (iii) 정규식으로 SELECT/ASK/DESCRIBE/CONSTRUCT 외 거부.

## 의존성
- `mcp` (Python MCP SDK) — 없으면 서버는 경고만 찍고 skeleton 모드로 동작
- `python-frontmatter`, `pyyaml`
- `rdflib` — sparql / ontology 파싱
- 테스트: `pytest`

## 테스트
- `tests/test_obsidian_mcp.py` 가 `tests/fixtures/obsidian_vault/` 를 사용해
  read / list / search / write(dry-run) / sparql 을 검증한다.

## 설계 메모
- tools.py 는 **MCP SDK 에 의존하지 않는다** — 순수 함수이므로 단위테스트 용이
- server.py 에서만 `mcp` 의존성 import → 실행환경에만 SDK가 있으면 된다
- SDK 미설치 시 `server.py --help` 가 친절한 설치 안내를 출력
