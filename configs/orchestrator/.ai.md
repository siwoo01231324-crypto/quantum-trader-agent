# configs/orchestrator/

## 목적
오케스트레이터 전략 배선 구성 파일. 어떤 전략을 어떤 파라미터로 `AsyncStrategyOrchestrator` 에 등록할지를 **정적 YAML** 로 선언한다.

## 구조

- `production.yaml` — 프로덕션 전략 목록. `src/portfolio/config_loader.py` 의 `load_orchestrator_from_yaml()` 이 읽어 `AsyncStrategyOrchestrator` 인스턴스를 생성한다.

## YAML 스키마

```yaml
strategies:
  - id: <strategy_id>          # register_strategy 에 쓰이는 고유 식별자
    class: <dotted.import.path> # importlib 로 로드할 클래스 경로
    kwargs:                     # 클래스 생성자에 전달할 keyword 인자
      metalabeler:              # (선택) MetaLabeler 주입 시 사용
        load_path: <dir>        # MetaLabeler.load(Path(load_path)) 로 치환
      metalabeler_threshold: 0.5
```

## 로더 연결

`src/portfolio/config_loader.py` — `load_orchestrator_from_yaml(path, policy)`:
1. YAML 파싱 → strategy 목록 순회
2. `importlib` 로 `class` 동적 로드
3. `kwargs.metalabeler.load_path` 가 있으면 `MetaLabeler.load(Path(...))` 결과로 치환 (없으면 pass-through)
4. `_StrategyAdapter` 로 감싸 `register_strategy(id, adapter)` + `register_strategy_returns(id, pd.Series(dtype=float))` 호출

## 운영 규칙

- **LLM 이 kwargs 값 결정 금지** — CLAUDE.md 불변식 #6. 이 파일은 운영자가 직접 편집한다.
- **롤백** = `momo-btc-v2-meta` 블록을 주석 처리하는 단일 파일 변경.
- 새 전략 추가 시 `id` 는 레포 전체에서 고유해야 한다. 중복 시 `config_loader` 가 `ValueError` 를 발생시킨다.
