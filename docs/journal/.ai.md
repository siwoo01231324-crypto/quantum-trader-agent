# docs/journal/

## 목적
일별 거래 리포트를 저장한다. 자동 계좌(cs-tsmom 등)와 수동 계좌의 체결 내역,
진입 근거, 결과 분석, 다음 날을 위한 교훈을 기록한다.

## 구조
- 파일명: `YYYY-MM-DD.md` (KST 기준 날짜)
- frontmatter type: `trading-journal` (Obsidian 볼트 커스텀 타입, CI warn 허용)
- 소스 데이터: `docs/journal_data/YYYY-MM-DD.json` 에서 자동 생성

## 역할
- 거래 결과를 구조화된 형태로 기록해 패턴 학습 및 전략 개선에 활용
- auto_fills / auto_signals 는 strategy spec 의 backtest 기대값과 대조
- manual_trades 는 사용자 note 를 인용해 진입 근거를 보존
- cs_tsmom_top10 은 유니버스 상태 스냅샷으로 참조용

## 생성 흐름
```
docs/journal_data/YYYY-MM-DD.json
  → (claude/journal-YYYY-MM-DD 브랜치에서 자동 작성)
  → docs/journal/YYYY-MM-DD.md
  → PR (auto-merge 없음, 사용자 리뷰 후 머지)
```

## 주의
- `.draft.md` 불필요 — 리포트는 PR 리뷰 후 그대로 머지
- 모든 PnL 수치는 JSON 원본 그대로 기록 (가공·환산 없음)
- 테스트 거래 항목(symbol="테스트")은 그대로 기록하되 분석 섹션에 명시
