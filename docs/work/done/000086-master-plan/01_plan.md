## 0. 이 기획서가 끝났을 때의 모습 (산출물 정의)

외부인이 백서 한 편을 읽으면 다음 5가지를 한 번에 답할 수 있어야 한다.

1. **이게 누구를 위한 무엇인가** — 한국 개인 투자자가 자기 PC 에 설치해 운영하는, 매매 결정을 사람이 아닌 규칙·통계·작은 인공지능 모델에 맡기는 자동매매 프로그램.
2. **무엇을 사고파는가** — 비트코인 선물(해외)과 한국 우량주(국내) 두 시장을 동시에 다룬다. 어느 종목을 후보로 삼고, 어느 종목은 제외하는지 기준이 명시.
3. **어떻게 매매를 결정하고 얼마나 베팅하는가** — 가격·거래량 같은 시장 신호로 1차 결정을 만들고, 작은 인공지능 모델(2차 필터)이 이걸 거를지 통과시킬지 결정. 베팅 크기는 변동성·승률 기반 자동 산출.
4. **무엇이 잘못됐을 때 어떻게 멈추는가** — 손실 한도·계좌 변동성 한도·연속 오류 같은 위험 신호가 켜지면 자동으로 신규 매수가 차단되거나 전체 청산이 실행되는 "비상정지" 장치가 어떻게 동작하는지.
5. **마지막에 어떻게 끝나는가** — 사용자가 단일 실행 파일을 켜고 모니터링 대시보드 한 화면을 보면서 매일 운영하는 모습. 이 단계까지 무엇이 남았는지 한 페이지 로드맵.

## 1. 시스템의 최종 모습 — End-to-End Vision (백서 §1 + §11 의 핵심)

> 백서를 읽는 사람이 "프로젝트가 마지막에 어떻게 돌아가는지" 한눈에 그릴 수 있도록 평어로 시나리오화. 이 절은 백서 §1(Executive Summary)·§11(로드맵) 의 토대.

### 사용자 운영 시나리오 (Phase 4 도달 시점)

1. 사용자는 자기 윈도우 PC 에 단일 실행 파일(qta.exe) 을 다운로드해 설치한다. 별도의 파이썬·도커 설치는 필요 없다.
2. 첫 실행 시 자기 증권계좌(한국 KIS)와 거래소(Binance) API 키를 입력하면, 키는 윈도우의 보안 저장소에 암호화되어 보관된다. 평문 파일에 절대 남지 않는다.
3. 실행 파일이 켜지면 백그라운드에서 시세 수신·신호 계산·주문 결정·리스크 검사·체결 기록 까지의 다섯 단계가 매초 자동으로 돈다.
4. 사용자는 브라우저로 `localhost:3000` 에 들어가 한 화면짜리 대시보드를 본다. 거기엔 (a) 현재 손익 그래프, (b) 보유 포지션과 한도 사용률, (c) 어떤 전략이 어떤 종목에 매수/매도 신호를 냈는지, (d) 비상정지 장치의 현재 상태가 한눈에 보인다.
5. 자동매매가 위험 한도 (예: 운용 자본 대비 5% 손실)에 닿으면 시스템이 스스로 멈추고, 텔레그램으로 사용자에게 알림이 가며 신규 매수가 막힌다. 보유 포지션 청산은 사용자 승인을 받아 별도 명령으로 실행한다.
6. 매주 일요일 자정에 시스템은 지난 주 거래 데이터를 모아 인공지능 모델(2차 필터)을 자동으로 다시 학습한다. 모델이 시장 변화에 적응하도록 한다.
7. 매월 1일에 시스템은 지난 달 손익 보고서·세금 신고용 매매내역 CSV 를 자동 생성해 사용자 PC 에 저장한다.
8. 새 버전이 나오면 시스템이 자체 업데이트 가능 여부를 알리고, 사용자 승인 후 검증된 패키지로 교체된다.

### 이 시나리오에 도달하기까지 남은 거리 (2026-04-26 기준)

| 단계 | 무엇이 가능해야 하는가 | 현재 상태 | 다음 작업 |
|------|----------------------|----------|----------|
| Phase 0 — 백테스트 | 과거 데이터로 전략이 의미 있는 수익을 내는지 검증 | 완료 | — |
| Phase 1 — 그림자 운영 (Shadow Paper) | 실시간 시세를 받지만 가짜 돈으로 매매 | 완료 (#80 머지) | 30 거래일 무사고 누적 |
| Phase 2 — 실제 모의계좌 운영 | 증권사 모의계좌 API 로 실제 주문 흐름 재현 | 미착수 | #105 |

## 7. Guardrails

### 반드시 지킨다 (Must Have)
- 모든 위키링크는 실제 노트 id 와 일치 (CI strict 통과 필수).
- 프론트매터 스키마 정확 — `type: whitepaper`, `id: qta-master-plan-v01`, `name`, `version: "0.1"`, `owner: siwoo`, `created: 2026-04-26`.
- 각 섹션은 위키링크 + 구체 수치 둘 다 포함 (이슈 AC 2번).
- 평어 우선 — 기술·수학 용어는 괄호 보조표기로만 (예: "꼬리 손실 평균 (CVaR)").
- 사용자 결정 미수령 항목은 임의로 채우지 않고 `> 결정 보류 — TBD` 명시.
- AGENTS.md "핵심 문서 링크" 섹션에 항목 추가.
- `docs/whitepaper/.ai.md` 신규 작성.
- §0/§1/§11 에 "프로젝트 최종 모습" 시나리오 명시 (사용자 요구사항 반영).

### 절대 하지 않는다 (Must NOT)
- 자동 커밋 / 푸시 (CLAUDE.md "행동 규칙" — 사용자 확인 필수).
- 미존재 노트로 위키링크 (CI strict 차단).
- LLM 이 주문 실행·리스크 결정 권한을 갖는 표현 (CLAUDE.md 불변식 6 위반).
