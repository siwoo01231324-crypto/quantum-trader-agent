# docs/

## 목적
프로젝트 문서의 단일 진실 공급원(SOT). 코드보다 문서가 먼저 — 기획·명세·작업 내역을 여기서 관리한다.

`docs/` 는 **Obsidian 볼트**로 열 수 있다 (issue #47). `[[위키링크]]` · 그래프뷰 · Dataview 쿼리 · 도메인 온톨로지(RDF/Turtle) 기반 추론 지원.

## 구조
| 디렉토리 | 역할 |
|----------|------|
| `.obsidian/` | Obsidian 볼트 설정 (app / core-plugins / community-plugins / graph) |
| `background/` | 배경 자료 (공모전 가이드, 시장 조사, 경쟁사 분석 등) |
| `whitepaper/` | 기술 백서 및 심층 기술 문서 |
| `specs/` | 기능 명세 + AC (Acceptance Criteria) |
| `onboarding/` | 환경 설정·기여 가이드 |
| `runbooks/` | 운영·장애 대응 런북 |
| `schemas/` | 노트 타입별 프론트매터 스키마 (Phase 2에서 생성) |
| `ontology/` | 도메인 온톨로지 TTL + SPARQL 쿼리 (Phase 3에서 생성) |
| `dashboards/` | Dataview 대시보드 (Phase 4에서 생성) |
| `work/active/` | 현재 진행 중인 이슈별 작업 내역 |
| `work/done/` | 완료된 이슈 작업 내역 |

## 규칙
- `work/active/` 폴더명 형식: `{PADDED}-{짧은이름}` (예: `000015-my-feature`)
- `work/done/` 으로의 이동은 `/finish-issue` 또는 `/drop-issue` 커맨드가 자동 처리
- `DROPPED-` 접두사가 붙은 폴더는 중도 포기된 이슈
- 모든 하위 디렉토리에 `.ai.md` 포함 필수

## Obsidian 볼트 사용
- obsidian.md 설치 → "Open folder as vault" → `docs/` 선택
- 필수 커뮤니티 플러그인: Dataview, Templater (볼트 첫 오픈 시 설치 알림)
- 설치 가이드: [onboarding/obsidian-setup.md](onboarding/obsidian-setup.md) (Phase 5 예정)
- 그래프뷰 색상 규칙: 전략=파랑, 신호=하늘, 리스크=빨강, 종목=주황, 백테스트=보라, 인시던트=주홍, 온톨로지=연보라
