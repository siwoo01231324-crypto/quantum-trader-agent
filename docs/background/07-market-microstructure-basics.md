# 주식 시장 구조 기초: 호가·체결·유동성·KRX 특성

> Phase 1 기초 문서. 전략·주문 시스템 설계의 공통 전제.

---

## 1. 호가창(Order Book) 구조

호가창은 매도 측과 매수 측의 미체결 주문을 가격별로 쌓아 보여주는 테이블이다. KRX 정규 시장에서는 10단계 호가(매도 1~10호가, 매수 1~10호가)가 공개된다.

| 구분 | 가격 | 수량 |
|------|------|------|
| 매도 3호가 | 50,300 | 1,200 |
| 매도 2호가 | 50,200 | 800 |
| 매도 1호가 | 50,100 | 500 |
| **최우선 매수** | **50,000** | **300** |
| 매수 2호가 | 49,900 | 600 |
| 매수 3호가 | 49,800 | 1,500 |

- **매도 1호가(Ask)**: 시장에서 가장 낮은 매도 희망가
- **매수 1호가(Bid)**: 시장에서 가장 높은 매수 희망가
- **스프레드**: Ask − Bid = 호가 간 차이

## 2. 매칭엔진과 체결 프로세스

### 2-1. 호가 우선순위 3원칙

KRX 매칭엔진은 다음 순서로 주문을 처리한다.

| 순위 | 원칙 | 내용 |
|------|------|------|
| 1 | **가격 우선** | 매도는 저가일수록, 매수는 고가일수록 우선 체결 |
| 2 | **시간 우선** | 동일 가격이면 먼저 접수된 주문이 우선 (동시호가 제외) |
| 3 | **수량 우선** | 동일 가격·시간이면 수량이 많은 주문이 우선 배분 |

### 2-2. 접속매매(연속 단건 매매)

정규시장(09:00~15:30)에서 적용되는 기본 체결 방식이다. 매도·매수 호가가 교차(매수호가 ≥ 매도호가)하는 순간 즉시 체결되며, 복수 가격으로 거래가 성립한다.

1. 투자자가 주문 제출 → 증권사 HTS/API → KRX 거래 시스템(EXTURE+)
2. 매칭엔진이 가격·시간·수량 우선원칙으로 호가 매칭
3. 체결 시 매수자·매도자에게 체결 통보 및 결제 큐 적재
4. T+2일 결제(증권 인도·대금 지급)

### 2-3. 단일가매매(동시호가)

특정 시간대에 호가를 일괄 접수한 뒤 하나의 가격으로 체결하는 방식. 이 시간대에는 **시간 우선 원칙이 배제**되고 가격·수량 우선만 적용된다.

| 시간대 | 단일가매매 적용 구간 |
|--------|----------------------|
| 장 개시 전 | 08:30~09:00 (개장 동시호가) |
| 장 마감 | 15:20~15:30 (마감 동시호가) |
| 변동성완화장치(VI) 발동 시 | 발동 후 2분간 단일가 전환 |
| 서킷브레이커 재개 시 | 거래 재개 후 10분간 단일가 |

## 3. 유동성·스프레드·슬리피지

### 3-1. 시장 유동성(Liquidity)

유동성은 "원하는 가격에 원하는 수량을 즉시 거래할 수 있는 능력"이다. 유동성이 높을수록 호가창이 두텁고, 대규모 주문에도 가격 충격이 작다.

- **대형주(삼성전자 등)**: 유동성 높음 → 스프레드 좁음, 슬리피지 낮음
- **소형주·저유동성 종목**: 유동성 낮음 → 스프레드 넓음, 슬리피지 높음

### 3-2. 스프레드(Bid-Ask Spread)

```
스프레드 = 매도 1호가(Ask) − 매수 1호가(Bid)
스프레드율(%) = (Ask − Bid) / Ask × 100
```

스프레드는 시장 조성 비용이자 즉시 거래의 암묵적 비용이다. 시장가 주문을 넣으면 최소한 스프레드만큼은 불리한 가격에 체결된다.

### 3-3. 슬리피지(Slippage)

슬리피지는 **주문 제출 시점의 예상 가격과 실제 체결 평균가의 차이**다.

| 발생 원인 | 설명 |
|-----------|------|
| 호가창 깊이 부족 | 대량 주문 시 여러 호가 레벨을 소진하며 평균가 악화 |
| 시장 충격(Market Impact) | 주문 자체가 시장가격을 움직임 |
| 지연(Latency) | 주문 제출과 체결 사이 시간 동안 가격 변동 |

**예시**: 호가 50,000원에 10,000주 시장가 매수 주문을 냈으나, 50,000원에 3,000주, 50,100원에 4,000주, 50,200원에 3,000주가 체결되어 평균 체결가 50,090원 → 슬리피지 = 90원/주

## 4. KRX 특유 제도

### 4-1. 가격제한폭

KRX는 과도한 가격 급변을 막기 위해 1일 가격 변동 범위를 제한한다.

| 시장 | 가격제한폭 | 시행 시기 |
|------|-----------|-----------|
| 유가증권시장(KOSPI) | ±30% | 2015년 6월 15일~ |
| 코스닥(KOSDAQ) | ±30% | 2015년 6월 15일~ |
| 코넥스(KONEX) | ±30% | 동일 |

- **상한가**: 전일 종가 기준 +30% 상승 한도
- **하한가**: 전일 종가 기준 −30% 하락 한도
- ETF·ETN·파생상품은 별도 제한폭 규정 적용

### 4-2. 가격 안정화 장치 비교

| 장치 | 발동 조건 | 효과 |
|------|-----------|------|
| **변동성완화장치(VI)** | 종목이 참조가 대비 ±10% 초과 변동 | 2분간 단일가매매 전환 |
| **서킷브레이커 1단계** | KOSPI 전일비 −8% 이상 | 20분 거래 중단 후 10분 단일가 재개 |
| **서킷브레이커 2단계** | KOSPI 전일비 −15% 이상 | 20분 거래 중단 후 10분 단일가 재개 |
| **서킷브레이커 3단계** | KOSPI 전일비 −20% 이상 | 당일 거래 종료 |
| **사이드카** | 선물 가격 ±5% 이상 1분 지속 | 5분간 프로그램 매매 주문 효력 정지 |

### 4-3. 거래 시간 전체 구조

| 구분 | 시간 | 방식 |
|------|------|------|
| 장 개시 전 시간외종가 | 08:00~08:30 | 전일 종가로만 체결 |
| 개장 동시호가 | 08:30~09:00 | 단일가매매 |
| 정규시장(접속매매) | 09:00~15:20 | 연속 개별 경쟁매매 |
| 마감 동시호가 | 15:20~15:30 | 단일가매매 |
| 시간외종가 | 15:40~16:00 | 당일 종가로만 체결 |
| 시간외단일가 | 16:00~18:00 | 10분 단위 단일가매매, ±10% 제한 |

### 4-4. 거래세 및 양도소득세

#### 증권거래세(Securities Transaction Tax)

매도 시 자동 부과되며, 증권사가 원천징수한다.

| 시장 | 세율 |
|------|------|
| 유가증권시장(KOSPI) | 0.05% |
| 코스닥(KOSDAQ) | 0.20% |
| 코넥스(KONEX) | 0.10% |

> 참고: 증권거래세법상 법정 세율은 0.35%이나, 시행령으로 시장별 탄력세율이 적용된다.

#### 양도소득세(Capital Gains Tax)

국내 상장 주식의 경우 일반 개인투자자는 **원칙적으로 비과세**이며, 다음 경우에만 과세된다.

| 과세 대상 | 세율 |
|-----------|------|
| 대주주(종목별 지분 1% 또는 시가총액 10억원 이상 보유) 양도 | 20~25% (보유기간·금액에 따라 차등) |
| 대주주 1년 미만 보유(중소기업 外) | 30% |
| 중소기업 주식 양도(일반) | 10% |

- 기본공제: 연간 250만원
- 신고: 양도일 속한 반기 종료 후 2개월 내 예정신고

### 4-5. 시간외 거래 특성 요약

| 항목 | 시간외종가 | 시간외단일가 |
|------|-----------|--------------|
| 시간 | 15:40~16:00 | 16:00~18:00 |
| 가격 범위 | 당일 종가 고정 | 당일 종가 ±10% 이내 |
| 체결 방식 | 즉시 체결(수량 우선) | 10분 단위 단일가 |
| 유동성 | 낮음 | 매우 낮음 |
| 주요 용도 | 장 마감 후 종가 기준 거래 | 공시 대응, 야간 정보 반영 |

## 5. 퀀트 시스템 설계 함의

1. **슬리피지 모델링 필수**: 소형주·저유동성 종목은 호가창 깊이를 시뮬레이션해야 한다.
2. **단일가 구간 주의**: 개장·마감 동시호가 및 VI 발동 시 접속매매 가정이 깨진다.
3. **거래세는 매도 비용**: 전략 수익률 계산 시 매도마다 세율 적용 필수.
4. **가격제한폭 ±30%**: 하루 최대 손실/이익의 하드 캡. 극단적 갭 리스크 시나리오에서도 이 범위를 벗어나지 않는다.
5. **시간외거래 유동성 희박**: 알고리즘 주문을 시간외 구간에 배치할 경우 스프레드·슬리피지가 대폭 확대된다.
