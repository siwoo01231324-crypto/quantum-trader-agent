# [외부 참고] Snowflake Marketplace 대체데이터 × 주식 투자 지표 데모

> **출처**: Snowflake 해커톤 2026 UGM (2026-03-17), 발표자 **박준호 (스튜디오)**
> 제목: "아이디어에서 구현까지 단숨"
> 영상 URL: https://snowflake.wistia.com/medias/80p2bxdtee
> 전체 분량: 약 15분
> 작성일: 2026-04-13 (트랜스크립트 기반 요약)

---

## 1. 핵심 한 줄

**"남들이 보지 않는 지표"를 찾기 위해 백화점 방문자수·카드소비 같은 대체데이터를 Snowflake Marketplace로 가져오고, 주가와 결합해 Streamlit 대시보드 + 백테스팅 + LLM 투자 인사이트 앱을 만든 사례.**

---

## 2. 배경 & 페인포인트

- 발표자는 **개인 주식 투자**를 하면서 "이동평균선·RSI처럼 남들 다 보는 지표 말고 **남들이 안 보는 지표**가 없을까?"를 고민.
- 2025년 4월 Snowflake 해커톤에서 **Streamlit 기반 데이터 앱** 요구사항 + **Loplat / 데이터노스 / SPH 세 업체의 데이터**를 활용하라는 과제가 나옴.
- 세 업체 중 두 가지 데이터가 눈에 띔:
  1. **Loplat**: 서울시 Top 3 백화점(신세계 강남, 더현대 서울, 롯데 본점) **방문자수** (위치기반)
  2. **SPH**: 해당 백화점 **신한카드 소비내역**

---

## 3. 가설

> 백화점 방문자수 또는 소비금액이 늘어나면 → **해당 백화점 관련 주식의 주가가 상승할 확률이 높다.**

- 반박 가능성도 열어두고 실증 검증(백테스팅)으로 접근.

---

## 4. 개발 파이프라인 (5단계)

```
입수 → 정제 → 분석 → 시각화 → AI
```

### 4.1 데이터 입수 — Snowflake Marketplace
- **클릭 한 번**으로 Loplat/SPH 데이터를 자기 DB에 연동 → 적재까지 자동.
- "데이터를 어떻게 퍼싱·저장하지?" 같은 엔지니어링 병목이 사라짐.
- 주가 데이터는 **공공데이터포털 API**로 별도 수집.
- 해커톤 제약으로 전종목이 아닌 **백화점별 관련 종목 4개씩 × 3사 = 12종목**만 선정.
- 컬럼: 일자, 시가, 고가, 저가, 종가, 거래량.

### 4.2 정제
- 결측치·이상치·데이터 형식·스키마 검증·품질 검사.
- 핵심 원칙: **"Garbage in, Garbage out"** — AI 분석이 뛰어나도 입력이 오염되면 무의미.
- Snowflake 내부에서 **dbt**와 통합 사용 가능(이 데모에서는 시간 제약으로 생략).
- 실시간 연동이 아닌 과거 데이터 백테스팅용이라 1회성 검증만 수행.

### 4.3 분석
- Snowflake 콘솔에서 DDL/쿼리 직접 실행.
- 최종 테이블 3종:
  1. **방문자 트렌드 테이블**
  2. **소비 트렌드 테이블**
  3. **주가 테이블**

### 4.4 시각화 — Streamlit in Snowflake
- Snowflake에 내장된 Streamlit 기능(코드 몇 줄로 인터랙티브 앱).
- 왼쪽 사이드바 메뉴 + 그래프 + 12종목 주가 표시.

### 4.5 AI 분석
- 당시(2025) **Claude 3.5**를 Snowflake에서 호출.
- 현재는 4.6까지 나와 있고 Snowflake Cortex에서도 최신 모델 사용 가능(발표자 언급).

---

## 5. 데모 앱 기능 (대시보드 구조)

| 페이지 | 내용 |
|--------|------|
| 소비/방문 현황 | 월별 추이 그래프 |
| 주가 현황 | 12종목의 일봉·월별·주가 등락률·거래량 |
| **백테스팅 페이지** | 방문자 증가율·소비 증가율 임계값 선택 → 매수 시점·보유 기간·성과 자동 계산 |
| **수익률 비교** | 단기(1~3개월) vs 장기(6~12개월), Box plot 으로 4분위수 + 결측치 표시 |
| **LLM 투자 인사이트** | 백화점·종목 선택 후 "이 종목 투자해야 할까?" 자연어 질의 → 백테스팅 결과 + 시장 상황을 backend에서 합성해 전략 요약(권고 + 주의점) 반환 |

### 백테스팅 예시
- **더현대 서울**: 2023년 11월 전월 대비 방문자수 +17%
- 그 달 종가로 **현대홈쇼핑** 매수 → 4개월 뒤 **+47%** 성과.

---

## 6. 전략 & 결과

### 전략 정의 (실전형)
- **매수 시그널**: 전월 대비 방문자수 +10% 이상
- **손절**: -2%
- **익절**: +4% 도달 시 절반 매도 → 나머지는 부분 이익 실현 (최고점 매도는 불가)
- **투자 기간**: 1년
- **초기 자본**: 1,000만 원 기준 백테스팅

### 결과
- 3개 백화점 모두 **손절 구간 존재**, 최종 +10% / +12% / +20% 수익.
- **같은 기간 KOSPI -30% 하락장**이었음을 감안하면 상대적 초과 성과.
- 즉, **하락장에도 대체데이터 기반 신호가 절대 수익을 가능케 함**이 실증됨(표본은 제한적).

---

## 7. 회고 & 확장 아이디어 (발표자 구상)

- **BM 후보**: 데이터 제공업체와 협약 → Snowflake 내부에서 정식 데이터 수령 → Queue/이벤트 시스템을 통해 **카카오톡·텔레그램 알림 API** / **뉴스레터** 서비스화.
- Snowflake는 AWS/GCP와의 통합성이 좋아 외부 송신 확장이 용이.
- 개인 일정으로 실제 제품화는 미진행, 재개 의향 언급.

---

## 8. 해커톤 준비 팁 (발표자 관점)

1. **페인 포인트부터** — 보상(상금)보다 "무엇을 해결할지"가 출발점.
2. **도구는 부차적** — 크리덴셜 도구든 Cortex든, 어떤 문제에 어떤 임팩트를 주느냐가 핵심.
3. Snowflake **Cortex** (Claude 유사 코드 도구)가 플랫폼 내부 통합·보안 측면에서 유리, 활용 권장.

---

## 9. 본 프로젝트(quantum-trader-agent) 적용 가능성

> 핵심 결정: 본 프로젝트는 **한국 개인의 저빈도 규칙기반 퀀트 자동매매**(#5 positioning 결론). 아래 요소가 직접 연결됨.

| 영역 | 적용 포인트 | 연결 이슈 |
|------|------------|-----------|
| **대체데이터** | 백화점 방문자수·카드소비 같은 시장 외부 신호를 알파 팩터 후보로 카탈로그화 | #23 피처·알파 소스 카탈로그 |
| **데이터 파이프라인** | Snowflake Marketplace 방식 = "원클릭 연동" 레퍼런스. 자체 구축 시 Parquet+DuckDB로 대체 가능하나 API 계약 문서는 참고 가치 큼 | #20 데이터 레이크 스키마 |
| **백테스팅 앱** | Streamlit in Snowflake 대신 **로컬 Streamlit + Backtrader/LEAN** 스택으로 동일 UX 재현 가능 | #21 백테스트 엔진 선택 / #22 검증 프로토콜 |
| **전략 템플릿** | 시그널(알파) → 리스크 룰(2% 손절 / 4% 익절 절반) → 부분 청산. 본 프로젝트 **리스크 룰 DSL** 예시 정책 "보수"에 그대로 매핑 가능 | #24 리스크 룰 DSL |
| **LLM 인사이트 레이어** | 백테스팅 수치·시장 상황을 자연어 요약하는 "투자 인사이트 에이전트"는 우리 LLM 에이전트 레이어 탐색의 구체 예시 | #30 LLM 에이전트 레이어 |
| **알림 BM** | 카카오톡·Telegram·뉴스레터로 시그널 배포. Kill Switch/관측성 알림 채널과 재사용 가능 | #26 관측성 스택 / #27 Kill Switch |

### 구체적 차용 제안
1. **대체데이터 1~2개 POC**: 공공데이터·Finance Data Reader로 얻을 수 있는 대체 신호(예: 통신사 유동인구, 검색량)를 Phase 1 MVP에 optional factor로 추가.
2. **백테스팅 UI 컨벤션**: "임계값 선택 → 자동 계산 → Box plot 장단기 비교"를 `docs/specs/execution-algorithms.md` 검증 UX 표준으로 채택.
3. **전략 기본 템플릿**: "시그널 + 2% 손절 + 4% 반익절"을 `policy-examples/` 첫 사례로 수록.
4. **LLM 인사이트 API**: 백테스팅 결과 + 현재 시세 + 전략을 프롬프트로 합성 → "권고 + 주의점" 포맷을 내부 도구로 시제품화.

## 10. 주의점 · 한계

- 표본 매우 제한적(12종목 × 1년).
- **데이터 라이선스**: Loplat/SPH는 해커톤 한정 제공. 상용 사용 시 B2B 계약 필요.
- 신한카드만 커버 → 카드 유저 편향.
- 백테스팅 시점(하락장)이 특수했음. 상승장에서 같은 전략이 알파를 낼지 재검증 필요.
- 한국 개인에게 실제 매수 알림만 전달하는 BM은 **투자자문업 인가** 문제 체크 필요.
