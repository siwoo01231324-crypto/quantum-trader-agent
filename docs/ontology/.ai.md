# docs/ontology/

## 목적
트레이딩 도메인 지식을 RDF/OWL 기반 온톨로지로 형식화한다. Obsidian 볼트 내 마크다운 프론트매터를 TTL 인스턴스로 동기화해 SPARQL 로 질의 가능한 지식 그래프를 구축한다.

## 구조
```
docs/ontology/
├── .ai.md                   ← 이 파일
├── trading.ttl              ← T-Box (클래스·프로퍼티 정의)
├── instances.ttl            ← A-Box (프론트매터에서 자동 생성, gitignore 여부는 정책)
└── queries/                 ← SPARQL 쿼리 모음
    ├── live_strategies.rq
    ├── critical_violations.rq
    └── strategy_without_tests.rq
```

## 역할
- `trading.ttl` : 7개 노트 타입(Strategy/Signal/RiskRule/Instrument/Backtest/Incident/MLModel)의 클래스·속성 정의
- `instances.ttl` : `scripts/ontology_sync.py --write` 로 생성
- `queries/*.rq` : 운영·분석용 SPARQL

## 사용법
```bash
# 프론트매터 → TTL 동기화
python scripts/ontology_sync.py --check    # 변경 감지만
python scripts/ontology_sync.py --write    # instances.ttl 재생성

# SPARQL 쿼리 실행 예시 (rdflib)
python -c "from rdflib import Graph; g=Graph(); g.parse('docs/ontology/trading.ttl',format='turtle'); g.parse('docs/ontology/instances.ttl',format='turtle'); print(list(g.query(open('docs/ontology/queries/live_strategies.rq').read())))"
```
