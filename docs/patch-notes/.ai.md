# docs/patch-notes/

## 목적
QTA 자동매매 시스템의 버전별 패치노트를 구조화 저장한다. 대시보드 `/patch-notes`
페이지가 본 디렉토리를 데이터 소스로 읽어 렌더링한다.

## 구조
- `index.yaml` — 단일 데이터 파일. `versions:` 리스트에 entry 추가 (newest first)
- 스키마는 파일 상단 주석 참조 (version/date/title/tags/summary/items/refs)

## 역할
- 사용자가 작업 내역을 시간순으로 추적 (브라우저 1탭)
- 자동매매 전략·필터·UI 변경 이력 단일 진실
- 대시보드 nav pill `패치노트` 에서 즉시 접근

## 추가 절차
1. `index.yaml` 의 `versions:` 리스트 **맨 앞에** 새 entry append
2. 버전은 semver 또는 자유 형식 (`v0.6.0`, `v0.5.x` 등)
3. tags: `feature|fix|perf|chore|docs|safety` 중 선택 (대시보드 색 입힘)
4. refs 는 관련 파일 경로 또는 GitHub PR URL
