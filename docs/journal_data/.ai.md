# docs/journal_data/

## 목적

매일 23:50 KST에 dashboard `/api/journal/export` 엔드포인트가 호출하는 일일 거래
데이터 스냅샷. Claude Code Routines (`docs/routines/cs-tsmom-daily-report.md`)가
매일 자정 이 파일을 읽고 `docs/journal/YYYY-MM-DD.md` 리포트를 생성한다.

## 파일 구조

```
docs/journal_data/
├── YYYY-MM-DD.json   # 날짜별 거래 데이터 스냅샷
└── .ai.md            # 이 파일
```

## JSON 스키마

```json
{
  "date_kst": "YYYY-MM-DD",
  "kst_window_start": "YYYY-MM-DDT00:00:00+09:00",
  "now_kst": "YYYY-MM-DDTHH:MM:SS+09:00",
  "counts": {
    "auto_fills": <int>,
    "auto_signals": <int>,
    "manual_trades": <int>,
    "airborne_fires": <int>
  },
  "auto_fills": [...],        // WAL 자동 체결 기록 (strategy_id 포함)
  "auto_signals": [...],      // 자동 전략 신호 이벤트 (reason 필드)
  "manual_trades": [...],     // /manual 폼 입력 수동 거래
  "cs_tsmom_top10": {...},    // cs-tsmom-crypto-daily 모델 시그널 점수표
  "airborne_fires": [...]     // qta-airborne-daemon 텔레그램 알림 전수
}
```

## 생성 방법

### 자동 (권장)
dashboard 에서 **"오늘 Export"** 버튼 클릭 (POST `/api/journal/export`).
git commit + push 까지 1-click 처리.

### 수동 (백필용)
```bash
python scripts/export_journal_for_date.py YYYY-MM-DD
```

## 주의사항

- `.json` 파일은 **수동 수정 금지** — 데이터 무결성 보장을 위해 dashboard API 경유만
- `auto_fills` 에 `strategy_id` 가 없는 레코드는 분석 제외 대상 (웹 수동 주문 또는
  WAL 파싱 미매핑 fill)
- `cs_tsmom_top10` 의 HOLD/ENTER/EXIT 는 모델 권고일 뿐 실제 체결 여부는
  `auto_fills` 가 진실
- `airborne_fires` 의 시뮬레이션(TP/SL/timeout) 은 dashboard 또는 별도 스크립트에서
  Binance USDM Futures 15m klines 를 fetch 해야 산정 가능
