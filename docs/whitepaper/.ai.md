# docs/whitepaper/

## 목적

QTA(Quantum Trader Agent) 프로젝트의 단일 source-of-truth **마스터 기획서** 보관. 외부 투자자·신규 합류자·미래의 자기 자신이 시스템 전체를 한 문서로 재현할 수 있도록 한다.

## 구조

| 파일 | 종류 | 설명 |
|------|------|------|
| `qta-master-plan-v01.md` | 정식 기획서 | v0.1 (Phase 0 완료, Phase 1 Shadow Paper 운영 시작 시점 스냅샷) |
| `_drafts/*.md` | 작성 보조 | 다관점 워커 초안. 정식 기획서 통합 후 보존(추적 목적) — Git 에는 커밋하지 않음 |

## 노트 정책

- **정식 노트**: `*.md` (확장자만). 프론트매터 `type: whitepaper` 필수.
- **초안**: `*.draft.md` 또는 `_drafts/` 하위. 정식 승격은 사람이 rename.
- 백서 버전 정책: `qta-master-plan-v{NN}.md` (v0.1, v0.2, ...). 새 버전은 추가 파일로 발행, 구버전 보존.
- 자동 commit/push 금지 — 정식 기획서는 사용자 리뷰 후 수동 커밋.

## 프론트매터 스키마

`docs/schemas/note-schemas.md` §12 참조. 필수 필드:

```yaml
---
type: whitepaper
id: qta-master-plan-v01    # 파일명과 일치
name: "QTA Master Plan v0.1"
version: "0.1"
---
```

권장 추가 필드: `owner`, `created`.

## 다른 디렉토리와의 관계

- `docs/specs/` — 개별 기능 스펙 (이 백서가 위키링크로 참조)
- `docs/background/` — 32개 리서치 노트 (백서 §2 에서 그룹별 인용)
- `docs/runbooks/` — 운영 절차 (백서 §10 에서 인용)
- `docs/work/done/` — 이슈별 작업 내역

## 새 버전 발행 시

1. 이전 버전 보존 (`qta-master-plan-v{N-1}.md` 그대로 유지).
2. 새 버전 `qta-master-plan-v{N}.md` 신규 작성.
3. `AGENTS.md` "핵심 문서 링크" 섹션 갱신 (최신 버전으로 포인터 변경).
4. CI strict 통과 확인: `python scripts/check_invariants.py --strict`.
