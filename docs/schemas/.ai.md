# docs/schemas/

## 목적
Obsidian 볼트 노트의 **프론트매터 규약**을 정의한다. CI 검증(`scripts/check_invariants.py`)과 온톨로지 변환(`scripts/ontology_sync.py`)의 입력 스펙.

## 구조
| 파일 | 역할 |
|------|------|
| `note-schemas.md` | 13개 타입 필드 정의: 엔티티(Strategy/Signal/RiskRule/Instrument/Backtest/Incident/PostMortem) + 문서(SpecArchitecture/Runbook/Research/Onboarding/Whitepaper/WorkDone) |

## 규칙
- 스키마 변경 시 반드시 마이그레이션 계획을 함께 작성
- 필드 추가는 선택(optional)으로 먼저 도입 → 안정화 후 필수(required) 승격
- CI 검증 단계는 **strict** 모드 (Issue #52 에서 전환 완료) — 위반 시 PR 차단
- 일괄 주입 도구: `scripts/migrate_frontmatter.py` (idempotent)
- 관련 이슈: #47 (온톨로지 도입), #52 (전 문서 프론트매터 일괄 적용 + CI strict)
