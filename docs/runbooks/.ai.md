# docs/runbooks/

## 목적
운영 런북 보관. 장애·수동 개입·정기 점검 절차를 단계별로 기술한다.

## 구조
- 파일명 형식: `{도메인}-runbook.md` (예: `kill-switch-runbook.md`)
- 각 런북은 트리거 조건 → 액션 단계 → 복구 체크리스트 구조를 따른다

## 현재 파일 목록
- `kill-switch-runbook.md` — Kill Switch 수동 발동·포지션 청산·복구 절차 (#27)

## 규칙
- 단계별 지시는 명확·실행가능하게 작성 (모호한 "모니터링하세요" 금지)
- 실제 사건 발생 후 런북을 회고·개정한다
