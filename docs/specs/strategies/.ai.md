# docs/specs/strategies/

## 목적
개별 거래 전략 명세 노트를 둔다. 각 파일 = 하나의 Strategy 노트 (`type: strategy`).

## 스키마
`docs/schemas/note-schemas.md` 의 **Strategy** 타입을 따른다.

## 파일명 규칙
- `{id}.md` (id 는 kebab-case, 예: `momo-btc-v2`)
- id 는 프론트매터 `id` 와 반드시 일치

## 대시보드 카탈로그 (#178)

`src/dashboard/strategy_catalog.py` 가 본 디렉토리의 `*.md` frontmatter 를 로드하여 `/strategies` 카드 그리드와 `/api/strategies` JSON 응답에 사용한다. 카드 표시용 선택 필드:

| 필드 | 타입 | 설명 |
|------|------|------|
| `sharpe_bt` | float\|null | 백테스트 Sharpe |
| `mdd_bt` | float\|null | 백테스트 최대낙폭 (음수, 예: -0.18) |
| `annual_return_bt` | float\|null | 백테스트 연수익률 (예: 0.24) |
| `backtest_period` | str\|null | "YYYY-MM-DD/YYYY-MM-DD" 형식 ISO 범위 |
| `last_updated` | ISO date | frontmatter 마지막 갱신일 |

값 모르면 `null`. 백테스트 실측 후 `src/backtest/frontmatter.py` 의 `update_strategy_frontmatter` 가 갱신.
