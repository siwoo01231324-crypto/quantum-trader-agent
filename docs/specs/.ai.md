# docs/specs/

## 목적
기능 명세와 AC(Acceptance Criteria) 문서를 보관한다. 구현 전에 작성하고, 이슈와 1:1로 연결한다.

## 구조
- 파일명 형식: `{이슈번호}-{기능명}.md` (예: `0015-user-auth.md`)
- 각 문서에는 배경·목표·AC·비고를 포함한다

## 현재 파일 목록
- `data-lake-schema.md` — OHLCV·호가·체결·팩터·메타 Parquet 스키마·파티셔닝·DDL·DuckDB (이슈 #20, 2026-04-13)
- `risk-rule-dsl.md` — YAML 6카테고리 한도 정책·pydantic 파서·보수/중립/공격 예시 (이슈 #24, 2026-04-13)

## 규칙
- AC는 구체적이고 검증 가능하게 작성한다
- 구현 완료 후 AC 달성 여부를 문서에 기록한다
