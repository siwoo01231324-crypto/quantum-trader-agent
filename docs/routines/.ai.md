# docs/routines/

## 목적

Claude Code Routines (Anthropic 클라우드 cron) 셋업 가이드 모음. 사용자가
[claude.ai/code/routines](https://claude.ai/code/routines) 또는 CLI `/schedule`
로 routine 등록 시 복사·붙여넣기할 prompt + 절차.

## 구조

| 파일 | 역할 |
|------|------|
| `cs-tsmom-daily-report.md` | 일일 거래 리포트 routine — 자동(cs-tsmom 등) + 수동 거래 대조 분석 → `docs/journal/YYYY-MM-DD.md` PR 생성 |

## 패턴

각 가이드 파일은 다음 구조 따름:
1. **frontmatter** `type: routine-guide`, `schedule:` 시각, `target: Claude Code Routines`
2. **비용** 섹션 — Max 구독 cover 여부 + token 추정
3. **데이터 source 옵션** — Routines 가 사용자 로컬 dashboard 접근 못 함 → repo 안 파일 / ngrok tunnel / 수동 export 중 하나
4. **등록 절차** — claude.ai/code/routines 또는 `/schedule` CLI
5. **Prompt 전문** — fence block 으로 복붙 가능
6. **trigger 시각** — KST 기준
7. **출력 형식** — markdown frontmatter + 본문 템플릿
8. **한계 + 후속** 솔직

## 핵심 의존성

- **Claude Code Routines** (2026-04-01 beta) — Pro/Max/Team/Enterprise 가능
- **dashboard `/api/journal/today`** (`src/dashboard/app.py`) — 통합 데이터 source
- **`docs/journal_data/`** — routine 이 읽을 일일 JSON snapshot (사용자가 dashboard 에서 export)
- **`docs/journal/`** — routine 이 생성하는 일일 리포트 markdown
- **Obsidian 볼트** — `docs/` 자체. routine 출력 PR 머지 → 자동 동기화

## 규칙

- routine 가이드 파일은 `type: routine-guide` frontmatter 필수 (`docs/schemas/note-schemas.md` 확장 후속)
- prompt 본문은 fence block 안에 — 사용자가 그대로 복사
- 모든 가이드는 **비용·한계 솔직히** 명시. 사용자가 운영 결정에 정확히 참고
