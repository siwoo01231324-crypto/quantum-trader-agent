# src/live/

## 목적

라이브 실행 프레임워크 (Phase 1 Shadow Paper). 실시간 WebSocket 시세 → orchestrator → PaperBroker → WAL/관측 파이프라인을 담당한다. 가상 체결(Paper) 단계이며 실자금 투입은 Phase 2(KIS 모의계좌) 이후다.

## 구조

현재 단계 (Phase C-1):

- `__init__.py` — 패키지 마커
- `types.py` — `OrderStatus`, `WALEvent`, `WALCorruption`, `Tick` 기반 타입
- `wal.py` — WAL write/replay + `WALWriteFailed`. fsync 성공 후 선택적 `observer(event)` 콜백 (#181, dashboard fan-out 용. 예외는 swallow + log warn)
- `process_lock.py` — 단일 프로세스 락 (filelock 기반)
- `schedule.py` (#216) — `wait_until_session_open(schedule, now_fn=, sleep_fn=)` async 헬퍼. `schedule='krx'` + 장 외 시간이면 다음 KRX 영업일 09:00 KST 까지 sleep, `'always'` / 장중이면 즉시 반환. `run_shadow_loop` 진입 직후 (ProcessLock 획득 전) 호출되어 마감 후 startup 시 KIS REST polling 폭주 (EGW00201) 를 사전 차단. `now_fn`/`sleep_fn` 주입으로 결정적 단위 테스트 가능.
- `conversion.py` — `intent_to_order_request` float→Decimal 변환 단일 지점
- `feed.py` — `MarketDataFeed` Protocol + `BinancePublicFeed` (USDT-M Futures public aggTrade WS)
- `feed_kis.py` (#177) — `KISMarketFeed` (REST 1m polling, `inquire-time-itemchartprice`) + `MockReplayFeed` (deterministic smoke). KRX 휴장 / 장외 시간 가드: `is_krx_market_open`. **#227 S5**: `stagger=True` 옵션 (cycle 내 N 호출을 `poll_interval/N` 간격으로 분산 — 350종 운영시 KIS REST 분당 한도 회피) + `max_qpm` 옵션 (round-robin window 로 budget 강제). 풀 KIS WS market data subscribe 는 별 이슈로 분리 (실 API key + tr_id `H0STCNT0` 필요).
- `strategy_position_store.py` (#192) — `StrategyPositionStore`. fill 이벤트 → `dict[strategy_id, dict[symbol, Decimal qty]]` 누적. `register_order` 로 client_order_id ↔ strategy_id 매핑 유지, `replay_from_wal` 로 부팅 시 복구, `ingest_fill_event` 로 `wal_observer` 와 결합. `DashboardState.position_provider = store.get_positions` 로 와이어링되어 `/api/strategies/{id}/toggle` OFF 시 그 전략 보유분만 청산.
- `pnl_aggregator.py` (#194) — `PnLAggregator`. fill 이벤트 → 평균매수가 (Decimal) 추적 + realized PnL 누적 (`realtime` / `daily` / `monthly` / `by_strategy` / `daily_for(sid)`). KST 09:00 business-date 기준으로 daily/monthly 분류 (자정 12시 X) — fill ts 검사 + property 호출 시 자동 reset. `replay_from_wal` 로 부팅 복구, `ingest_fill_event` 로 `wal_observer` 와 결합. `DashboardState.pnl_aggregator` 로 와이어링되어 `/api/pnl` + 카드 `pnl_today` 가 라이브 갱신. **#238**: `_apply_to_cost_basis` 가 SHORT 추적 — `sell` & held≤0 → 음수측 가중평균 진입가 (qty=0 0/0 가드: 미가드 시 broker zero-qty 정정 fill 이 aggregator 를 죽여 세션 P&L/리스크 게이팅 전체 정지), `buy` & held<0 → cover 실현 `(entry-cover)*qty`. 양방향 flip 처리 (short→long: `held>0 → avg=price`, long→short: `close_qty=min(qty,held)`, `held<0 → avg=price`). qty≤held LONG 매도 경로는 byte-identical legacy.
- `position_reconciliation.py` (#238 Item 5 — safe core) — `reconcile_positions(logical, broker_net, *, tol) -> list[ReconcileMismatch]` + `sum_logical_by_symbol`. **불변식**: Binance one-way Futures = 종목당 net 포지션 1개. `StrategyPositionStore` 의 per-strategy 논리 포지션은 거래소 실재가 아닌 *귀속*일 뿐 — Σ(logical/symbol) 을 broker 실 net 과 대조해 |delta|>tol 인 종목을 표면화 (incident: 논리 -0.90 vs Binance -0.85; live-scanner +0.05 "롱" 은 momo 숏에 netting 돼 실재 안 함). 순수·결정론. **귀속 재설계 자체(논리↔거래소 정합 회계)는 design-first 별도 — 이 모듈은 silent divergence 가시화 안전망**.
- `snapshot_builder.py` (#177) — `SnapshotBuilder.warmup()` (KIS REST backfill) + `build_snapshot(tick) -> dict` (전략용 ohlcv_history / history / factors). 멀티 timeframe pandas resample 기반. **#238 미배선 (사람 결정 필요)**: orchestrator 의 fraction→qty 변환(`portfolio.sizing.size_to_qty`)은 venue-correct 자본을 요구 — KRX 6자리는 `snapshot["equity_krw"]`, Binance `*USDT` 는 `snapshot["equity_usdt"]`. 현재 `build_snapshot` 과 `loop.py::_tick_to_market_snapshot` 은 `equity_krw` 를 placeholder(100000.0)로만 채우고 `equity_usdt` 키는 미설정. **#238 Item 9 배선 완료**: `SnapshotBuilder(balance_provider=...)` DI 추가 (default None → config placeholder → byte-identical 하위호환). 주입 시 `build_snapshot._inject_real_equity` 가 `provider.fetch()` (= `AccountInfoProvider`, Binance `available_usdt` + KIS `cash_balance`, 내부 15s 캐시) 로 `equity_usdt`/`equity_krw` 를 실값 overlay — fetch 예외/`ok=False` 면 placeholder 유지(hot-loop 비차단, 실패측은 Item-8 conversion 이 안전 drop). `loop.py:ShadowConfig.balance_provider` → `SnapshotBuilder` 전달, `scripts/live_run.py::_wire_balance_provider` 가 3 파이프라인(attached/dual-kis/dual-bnb/_run_pipeline) 모두 주입. 이로써 라이브 inert 해소 — 실잔고 기준 정상 사이징.
- `reconnect.py` — 지수 backoff + uniform jitter 재연결 정책 (`backoff_delay`, `with_reconnect`). #133 hotfix 이후 `loop.py::producer` 가 `backoff_delay` 직접 사용하여 WS keepalive timeout 시 자동 재연결 (max 100 attempts, 1s → 60s).

## 불변식

- **WAL-first**: 모든 주문/체결 이벤트는 append-only JSONL에 먼저 쓰고 메모리 반영. 쓰기 실패 시 주문 거부 + kill-switch trip.
- **Decimal 경계 단일 지점**: float→Decimal 변환은 `src/live/conversion.py::intent_to_order_request` 한 곳에서만 발생. `Decimal(float)` 직접 호출 금지.
- **Time source = 시스템 UTC**: WAL `ts`, 메트릭 ts 모두 `datetime.now(timezone.utc).isoformat()`. WS 서버 시간은 별도 `server_ts` 필드로 구분.
- **Windows SelectorEventLoop 정책**: `loop.py` 최상단에서 `WindowsSelectorEventLoopPolicy` 설정 (Phase C). ProactorEventLoop 금지.
- **단일 프로세스 락** (FMEA F9): `process_lock.py` 기반, 중복 실행 방지.
