# src/live/

## 목적

라이브 실행 프레임워크 (Phase 1 Shadow Paper). 실시간 WebSocket 시세 → orchestrator → PaperBroker → WAL/관측 파이프라인을 담당한다. 가상 체결(Paper) 단계이며 실자금 투입은 Phase 2(KIS 모의계좌) 이후다.

## 구조

현재 단계 (Phase C-1):

- `feed_kis.py` (#177) — `KISMarketFeed` (REST 1m polling, `inquire-time-itemchartprice`) + `MockReplayFeed` (deterministic smoke). KRX 휴장 / 장외 시간 가드: `is_krx_market_open`. **#227 S5**: `stagger=True` 옵션 (cycle 내 N 호출을 `poll_interval/N` 간격으로 분산 — 350종 운영시 KIS REST 분당 한도 회피) + `max_qpm` 옵션 (round-robin window 로 budget 강제). 풀 KIS WS market data subscribe 는 별 이슈로 분리 (실 API key + tr_id `H0STCNT0` 필요).
- `strategy_position_store.py` (#192) — `StrategyPositionStore`. fill 이벤트 → `dict[strategy_id, dict[symbol, Decimal qty]]` 누적. `register_order` 로 client_order_id ↔ strategy_id 매핑 유지, `replay_from_wal` 로 부팅 시 복구, `ingest_fill_event` 로 `wal_observer` 와 결합. `DashboardState.position_provider = store.get_positions` 로 와이어링되어 `/api/strategies/{id}/toggle` OFF 시 그 전략 보유분만 청산.
- `pnl_aggregator.py` (#194) — `PnLAggregator`. fill 이벤트 → 평균매수가 (Decimal) 추적 + realized PnL 누적 (`realtime` / `daily` / `monthly` / `by_strategy` / `daily_for(sid)`). KST 09:00 business-date 기준으로 daily/monthly 분류 (자정 12시 X) — fill ts 검사 + property 호출 시 자동 reset. `replay_from_wal` 로 부팅 복구, `ingest_fill_event` 로 `wal_observer` 와 결합. `DashboardState.pnl_aggregator` 로 와이어링되어 `/api/pnl` + 카드 `pnl_today` 가 라이브 갱신. **#238**: `_apply_to_cost_basis` 가 SHORT 추적 — `sell` & held≤0 → 음수측 가중평균 진입가 (qty=0 0/0 가드: 미가드 시 broker zero-qty 정정 fill 이 aggregator 를 죽여 세션 P&L/리스크 게이팅 전체 정지), `buy` & held<0 → cover 실현 `(entry-cover)*qty`. 양방향 flip 처리 (short→long: `held>0 → avg=price`, long→short: `close_qty=min(qty,held)`, `held<0 → avg=price`). qty≤held LONG 매도 경로는 byte-identical legacy. **per-venue split (통화혼합 버그 수정)**: scalar `_cum_realized`/`_daily`/`_monthly` 은 KRW(KIS)+USDT(Binance) 를 무의미하게 단순합산했었다. 모듈 레벨 `classify_venue(symbol)` — `*USDT`(len>4)→`"binance"`, 6자리 숫자 KRX→`"kis"`, 그 외→`"unknown"`(방어적, 절대 raise 안 함: live P&L/대시보드 공급원이라 raise 시 전 venue 회계 정지). `record_fill` 이 기존 누적 *옆에* `_cum_by_venue`/`_daily_by_venue`/`_monthly_by_venue` 버킷 추가 (`_apply_to_cost_basis` 불변). 신규 accessor `realtime_by_venue()` / `daily_by_venue()` / `monthly_by_venue() -> dict[str,float]`. 기존 scalar/`by_strategy`/`daily_for` 는 bit-identical 유지 (271+ 기존 테스트 + `test_pnl_aggregator.py` 그대로 green). per-venue daily/monthly 는 동일 `_refresh_business_window`(KST-09:00) reset 사용. 테스트 `tests/live/test_pnl_aggregator_by_venue.py`.
- `trade_history.py` (per-venue split 동반) — cross-run round-trip trade 재구성. 순수·결정론, I/O 는 주어진 WAL path 를 기존 `wal.replay` 로 읽는 것 뿐. `discover_wal_files(log_dir) -> list[Path]` (run 별 `*/wal.jsonl` glob, path 정렬; 디렉토리 부재 → `[]`). `reconstruct_trades(wal_paths) -> list[Trade]` — `order_filled` fill 을 (strategy_id, symbol) 별로 `pnl_aggregator._apply_to_cost_basis` 와 **동일한** long/short+flip 회계로 페어링 (entry open/증가, 반대측 reduce/close, oversized 반대측 = flip = close 후 신규 open). strategy_id 해석은 `StrategyPositionStore._resolve_strategy` / `PnLAggregator._resolve_strategy` 와 동일 — payload `strategy_id` 우선, 아니면 `client_order_id` 의 `{strategy}:{symbol}:{ts}:{idx}` prefix (해석 불가 fill 은 drop+warn). `Trade` 프로즌 dataclass: strategy_id/symbol/venue/side(long|short)/qty/entry_ts/entry_price/exit_ts/exit_price/realized_pnl(venue 통화, **cross-convert 금지**)/holding_seconds/status(closed|open). 미청산 잔량 → `status="open"` 1건 (exit/pnl=None). 정렬 = entry_ts → symbol 결정론. 테스트 `tests/live/test_trade_history.py`.
- `position_reconciliation.py` (#238 Item 5 — safe core) — `reconcile_positions(logical, broker_net, *, tol) -> list[ReconcileMismatch]` + `sum_logical_by_symbol`. **불변식**: Binance one-way Futures = 종목당 net 포지션 1개. `StrategyPositionStore` 의 per-strategy 논리 포지션은 거래소 실재가 아닌 *귀속*일 뿐 — Σ(logical/symbol) 을 broker 실 net 과 대조해 |delta|>tol 인 종목을 표면화 (incident: 논리 -0.90 vs Binance -0.85; live-scanner +0.05 "롱" 은 momo 숏에 netting 돼 실재 안 함). 순수·결정론. **귀속 재설계 자체(논리↔거래소 정합 회계)는 design-first 별도 — 이 모듈은 silent divergence 가시화 안전망**.
- `snapshot_builder.py` (#177) — `SnapshotBuilder.warmup()` (KIS REST backfill) + `build_snapshot(tick) -> dict` (전략용 ohlcv_history / history / factors). 멀티 timeframe pandas resample 기반. **#238 미배선 (사람 결정 필요)**: orchestrator 의 fraction→qty 변환(`portfolio.sizing.size_to_qty`)은 venue-correct 자본을 요구 — KRX 6자리는 `snapshot["equity_krw"]`, Binance `*USDT` 는 `snapshot["equity_usdt"]`. 현재 `build_snapshot` 과 `loop.py::_tick_to_market_snapshot` 은 `equity_krw` 를 placeholder(100000.0)로만 채우고 `equity_usdt` 키는 미설정. **#238 Item 9 배선 완료**: `SnapshotBuilder(balance_provider=...)` DI 추가 (default None → config placeholder → byte-identical 하위호환). 주입 시 `build_snapshot._inject_real_equity` 가 `provider.fetch()` (= `AccountInfoProvider`, Binance `available_usdt` + KIS `cash_balance`, 내부 15s 캐시) 로 `equity_usdt`/`equity_krw` 를 실값 overlay — fetch 예외/`ok=False` 면 placeholder 유지(hot-loop 비차단, 실패측은 Item-8 conversion 이 안전 drop). `loop.py:ShadowConfig.balance_provider` → `SnapshotBuilder` 전달, `scripts/live_run.py::_wire_balance_provider` 가 3 파이프라인(attached/dual-kis/dual-bnb/_run_pipeline) 모두 주입. 이로써 라이브 inert 해소 — 실잔고 기준 정상 사이징.
- `reconnect.py` — 지수 backoff + uniform jitter 재연결 정책 (`backoff_delay`, `with_reconnect`). #133 hotfix 이후 `loop.py::producer` 가 `backoff_delay` 직접 사용하여 WS keepalive timeout 시 자동 재연결 (max 100 attempts, 1s → 60s).

#238 Bug B — client_order_id 전략 귀속 보존:
- **루트원인**: `executor._make_key` 가 `{strategy_id}:{symbol}:{ts_ms}:{idx}` 반환 → 실 long 전략(`live-breakout-with-atr-stop`=27자)은 36자 Binance `newClientOrderId` 한도 초과 → adapter 가 regex 실패로 `generate(strategy="fallback")` 의 opaque sha256 을 *대신 제출* → 돌아온 fill 의 coid 를 `_resolve_strategy` 가 매핑 불가(`:` prefix 없음) → 전략별 포지션/거래이력/PnL 귀속 silent loss.
- **수정**: `_make_key` 가 `client_id.generate(strategy=strategy_id, symbol, side=f"{side}:{idx}", ts_ms)` 로 Binance-valid 36자 결정론 coid 를 *생성 지점에서* 발급(전략명 cram 불가하므로 prefix 포기, idx 는 side 컴포넌트에 fold 해 배치 내 유일성 유지, 동일 ms 재시도 → 동일 coid = idempotent). `execute_intents` 가 `place_order` *이전에* 그 정확한 coid 로 `position_store.register_order` 호출 (이전엔 ack 이후 `ack.client_order_id` 로 등록). adapter 는 이미 valid coid 를 그대로 유지 → **불변식: 등록 coid == 제출 coid == 돌아온 fill coid**. StrategyPositionStore explicit map 이 prefix 파싱 없이 해석. REJECTED ack 에 등록돼도 무해(map 엔트리는 fill 없이 포지션 안 만듦). PnLAggregator/trade_history 는 payload `strategy_id` 우선 경로 그대로(로직 불변). 테스트 `tests/live/test_executor_coid_attribution.py`.

## 불변식

- **WAL-first**: 모든 주문/체결 이벤트는 append-only JSONL에 먼저 쓰고 메모리 반영. 쓰기 실패 시 주문 거부 + kill-switch trip.
- **Decimal 경계 단일 지점**: float→Decimal 변환은 `src/live/conversion.py::intent_to_order_request` 한 곳에서만 발생. `Decimal(float)` 직접 호출 금지.
- **Time source = 시스템 UTC**: WAL `ts`, 메트릭 ts 모두 `datetime.now(timezone.utc).isoformat()`. WS 서버 시간은 별도 `server_ts` 필드로 구분.
- **Windows SelectorEventLoop 정책**: `loop.py` 최상단에서 `WindowsSelectorEventLoopPolicy` 설정 (Phase C). ProactorEventLoop 금지.
- **단일 프로세스 락** (FMEA F9): `process_lock.py` 기반, 중복 실행 방지.
