# src/backtest/swing/

## 목적
이슈 #99 — 스윙 전략 후보 (S1-S5 + iter 3 변형) 의 사전등록 백테스트.
각 전략은 순수 함수형이며 `Strategy` protocol 을 사용하지 않는다 (bench script 에서 position tracking).

## 구조
| 파일 | 역할 |
|------|------|
| `__init__.py` | 서브패키지 re-export |

| `atr.py` | Wilder ATR(N) 함수 (S2a trailing stop 에 사용) |
| `multi_exchange_carry.py` | F0-F5 multi-exchange funding arbitrage 전략 함수 (이슈 #174) |

## Regime Switching (#173, #199)
| 파일 | 역할 |
|------|------|
| `regime_switching.py` | R0-R6 variant router — regime 판별 후 S2c/S4 자동 전환 |

- R0: S2c always, R1: S4 always (baselines)
- R2: HMM-2state, R3: HMM-3state, R4: Threshold (4h, Sharpe 1.218), R5: Ensemble
- **R6: R4 logic on 1h bars with 4x retuned lookbacks (#199, Sharpe 1.201)** — return_lookback=720, entry_lookback=80, exit_lookback=40, vol_lookback=240
- `VARIANT_REGISTRY` dict로 frozen registry + sha256 witness
- `src/ml/regime/` HMM/Threshold 모듈 의존

## 규칙
- 전략 함수는 `shift(1)` 로 lookahead 방지 필수
- `Strategy` protocol 을 구현하지 않음 — bench script 전용 함수형 인터페이스
- S5 는 2-asset (BTC+ETH) 입력이 필요하므로 별도 시그니처
- S2c 는 `(signal, position_size)` tuple 을 반환 — bench script 에서 returns * pos_size 처리

## Paper Trading 어댑터 (#175, #143, #199)
- `paper_adapter.py` — `s2_donchian_voltarget` / `s4_funding_carry` / `route_r4` / `route_r6` 를 PaperBroker 로 연결하는 어댑터
  - `PaperAdapter.on_bar(df)` — bar 종료 시 호출, 신호 계산 → entry/exit order 제출
  - `AdapterConfig` — strategy, symbol, vol_target, return_lookback 등 설정
  - StrategyId: `s2c-voltarget` | `s4-funding` | `r4-switch` (4h, Sharpe 1.218) | `r6-switch` (1h, Sharpe 1.201, #199)
  - async-friendly: `AsyncBrokerAdapter` Protocol 호환
- `tests/test_paper_adapter.py` — 16개 단위 테스트 (R4 3 + R6 3 + state restoration 1 + 회귀 9)
- WAL 위치: `logs/shadow/{run_id}/wal.jsonl` (run_id = `phase1-{strategy}-{symbol}` 고정)
- 실행:
  - R4 (4h, Phase 1 #143 default): `python scripts/shadow_run_swing.py --strategy r4-switch --symbol BTCUSDT`
  - R6 (1h, #199 병렬 운영): `python scripts/shadow_run_swing.py --strategy r6-switch --symbol BTCUSDT`
  - `--interval` 미지정 시 자동 분류 (r4=4h, r6=1h)
