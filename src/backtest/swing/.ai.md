# src/backtest/swing/

## 목적
이슈 #99 — 스윙 전략 후보 (S1-S5 + iter 3 변형) 의 사전등록 백테스트.
각 전략은 순수 함수형이며 `Strategy` protocol 을 사용하지 않는다 (bench script 에서 position tracking).

## 구조
| 파일 | 역할 |
|------|------|
| `__init__.py` | 서브패키지 re-export |
| `strategies.py` | S1-S5 + iter 3 변형 signal 함수 (순수 함수, DataFrame -> Series) |
| `atr.py` | Wilder ATR(N) 함수 (S2a trailing stop 에 사용) |
| `multi_exchange_carry.py` | F0-F5 multi-exchange funding arbitrage 전략 함수 (이슈 #174) |

## 전략 매핑
| ID | 이름 | 학술 근거 | 참조 |
|----|------|-----------|------|
| S1 | TSMOM | Moskowitz/Ooi/Pedersen 2012 JFE | `docs/background/44-*.md` |
| S2 | Donchian Breakout | Turtle Trading 1983 | `docs/background/45-*.md` |
| S2a | S2 + ATR trailing stop | S2 + Wilder 1978 ATR(14), 2*ATR 거리 | iter 3 |
| S2b | S2 + hard stop/TP | S2 + -1% stop / +7% take-profit | iter 3 |
| S2c | S2 + vol-target sizing | S2 + annualized 15% vol target | iter 3 |
| S3 | EMA Pullback + RSI | Avellaneda/Lee 2010 + Wilder 1978 | `docs/background/46-*.md` |
| S4 | Funding Rate Carry | Perpetual futures basis | `docs/background/47-*.md` |
| S4a | Funding 양방향 | positive -> short, negative -> long | iter 3 |
| S5 | BTC-ETH Pairs | Log-ratio mean reversion | `docs/background/48-*.md` |

## 사전등록 무결성
- `scripts/bench_swing_strategies.py` (iter 2) 의 `VARIANT_REGISTRY` sha256 이 bench output 에 기록됨
- `scripts/bench_swing_iter3.py` (iter 3) 의 `VARIANT_REGISTRY` sha256 이 bench_output_iter3.json 에 기록됨
- 전략 파라미터 변경 시 registry hash 가 달라지므로 결과 추적 가능
- S4/S4a 는 `_funding_rate` 컬럼 부재 시 `DATA_UNAVAILABLE` 반환

## Regime Switching (#173)
| 파일 | 역할 |
|------|------|
| `regime_switching.py` | R0-R5 variant router — regime 판별 후 S2c/S4 자동 전환 |

- R0: S2c always, R1: S4 always (baselines)
- R2: HMM-2state, R3: HMM-3state, R4: Threshold, R5: Ensemble
- `VARIANT_REGISTRY` dict로 frozen registry + sha256 witness
- `src/ml/regime/` HMM/Threshold 모듈 의존

## 규칙
- 전략 함수는 `shift(1)` 로 lookahead 방지 필수
- `Strategy` protocol 을 구현하지 않음 — bench script 전용 함수형 인터페이스
- S5 는 2-asset (BTC+ETH) 입력이 필요하므로 별도 시그니처
- S2c 는 `(signal, position_size)` tuple 을 반환 — bench script 에서 returns * pos_size 처리

## Paper Trading 어댑터 (#175)
- `paper_adapter.py` — `s2_donchian_voltarget` / `s4_funding_carry` 를 PaperBroker 로 연결하는 어댑터
  - `PaperAdapter.on_bar(df)` — 4h bar 종료 시 호출, 신호 계산 → entry/exit order 제출
  - `AdapterConfig` — strategy, symbol, vol_target 등 설정
  - async-friendly: `AsyncBrokerAdapter` Protocol 호환
- `tests/test_paper_adapter.py` — 9개 단위 테스트 (entry/exit/sizing/s4-funding/round-trip)
- WAL 위치: `logs/shadow/{run_id}/wal.jsonl`
- 실행: `python scripts/shadow_run_swing.py --strategy s2c-voltarget --symbol BTCUSDT`
