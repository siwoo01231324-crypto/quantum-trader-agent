# src/backtest/

## 목적
경량 이벤트 기반 백테스트 엔진. Python 3.14 호환을 위해 Zipline 대신 pandas+numpy 기반 커스텀 엔진.

## 구조
- `protocol.py` — `Bar`, `Signal` 데이터클래스 (6 필드: action/size/reason + optional expected_return/win_probability/confidence) + `Strategy` runtime-checkable Protocol. **Signal 우선순위**: expected_return > rolling mu; win_probability → kelly_binary 라우팅.
- `engine.py` — bar-by-bar 이벤트 루프, 포트폴리오 추적, MDD halt
- `metrics.py` — Sharpe (daily return 기반), MDD, 총수익률, 승률 계산
- `bundle.py` — 파티션 Parquet 데이터 로더
- `frontmatter.py` — 전략 프론트매터 `sharpe_bt` 필드 업데이터
- `strategies/` — `Strategy` protocol 구현체 디렉토리

## 규칙
- Zipline/VectorBT 의존성 금지 (Python 3.14 비호환)
- Sharpe는 반드시 daily return 기반 `sqrt(365)` 사용
- `Strategy` protocol은 엔진 비의존적 — 향후 엔진 교체 가능
- **팩터 주입 컨벤션 (#71)**: 전략은 `required_factors: ClassVar[list[str]]` 클래스 속성을 선언하면 엔진이 바마다 `signals.registry.compute()` 로 계산해 `context["factors"][name]` 에 주입한다. Protocol 자체에는 선언하지 않는다(PEP 544 `ClassVar` 예외 + `@runtime_checkable` 호환성). 엔진은 `getattr(strategy, "required_factors", [])` 로 런타임 fallback. `context["factors"]` 는 엔진 예약 키 — 전략은 쓰지 말 것. `context` 자체는 바마다 새 dict (바 간 상태 보존 없음).
- **팩터 선계산 (#81)**: 엔진은 루프 진입 전 `required_factors` 의 팩터를 **전체 OHLCV 로 한 번** 계산하고, 바마다 `.iloc[:i+1]` 슬라이싱으로 `context["factors"]` 에 전달한다. 복잡도 O(N). 팩터 함수 자체는 수정하지 않으므로 full-recompute 경로와 bit-identical (테스트: `test_precompute_bit_identical_all_bars`). 70k bar 단일 RSI < 60s, 5팩터 < 120s. `momo-btc-v2` 는 이 훅을 소비한다 (`required_factors = ["rsi"]`).
- **비인과(non-causal) 팩터 금지**: `FactorSpec.causal=False` 로 등록된 팩터는 precompute 경로 진입 시 `AssertionError`. 기본값 `causal=True`. 라이브 스트리밍 엔진은 별도 `IncrementalFactorSpec` 이 필요할 것 (본 PR 범위 밖).
