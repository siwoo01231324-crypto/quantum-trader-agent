## 리스크 연동 (필수 — #70 이후)

모든 전략은 일수익률 시계열을 **포트폴리오 리스크 모듈로 공급** 해야 한다. 이 규칙은 선택이 아니라 **구현 계약** 이다.

### 어떻게
```python
from portfolio import StrategyOrchestrator
orch.register_strategy_returns("<strategy_id>", daily_return_series)
orch.refresh_portfolio_risk()
```

- `daily_return_series: pd.Series` — index=날짜, 값=그날 실현 수익률
- `strategy_id` = 전략 파일명 snake_case (예: `momo_btc_v2`)

### 왜
- `src/risk/portfolio.py` 의 LW shrinkage Σ / Meucci ENB / Historical CVaR 는 T×N 수익률 매트릭스를 입력으로 받음 ([[19-portfolio-risk]] §2.2, §3.1, §4.1).
- 전략이 수익률을 공급하지 않으면 `portfolio_risk is None` → 리스크 평가기가 **항상 ALLOW 경로** → 리스크 관리 무력화.
- 본 프로젝트는 **단일 evaluator 경로** (`risk.evaluate`) 유지 — 전략마다 자기 리스크 룰을 재구현하면 `rule_id` 라벨 공간 오염.
