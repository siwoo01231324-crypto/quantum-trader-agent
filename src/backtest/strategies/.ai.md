## 패턴 (2026-05-06 universe-scan default)

신규 전략은 두 패턴 중 명시적으로 선택한다 — 상세 규약은 `docs/specs/universe-scan-strategy-pattern.md`.

| 패턴 | 핵심 흐름 | 적용 전략 |
|------|----------|-----------|
| **universe-scan** (default) | 유니버스 → 랭킹 → 상위 N 보유, 주기적 리밸 | `breakout_donchian` (KOSPI200 → top-10), `cs_tsmom_kr_daily` (KOSPI200+KOSDAQ150 → top-20), `cs_tsmom_crypto_daily` (Binance top-30 → top-10), `cs_rsi_div_kr`, `cs_bb_macd_kr`, `cs_adx_ma_kr`, `cs_rsi_div_crypto`, `cs_macd_vol_crypto` (#218) |
| single-ticker (legacy) | 사전 지정 1~3 종목 + 단일 시그널 | `momo_btc_v2`, `momo_vol_filtered`, `meanrev_pairs`, `momo_kis_v1`, `swing_kr_daily` 4종 |

신규 전략을 single-ticker 로 추가하려면 spec 문서 (`docs/specs/strategies/<id>.md`) 의 "운영 규칙" 섹션에 사유 명시 (예: "특정 종목 펀더멘털 노이즈 hedge").

### Universe-scan strategies (#218)

- `_cs_helpers.py` — 공통 헬퍼: panel-form RSI / MACD / Bollinger / ATR / ADX, `build_weights` (cross-sectional 랭킹 → top-N → equal-weight rebal-aware ffill), `daily_returns_from_weights` (비용 차감), `liquid_mask_panel`.
- `cs_tsmom_kr_daily.py` — KRX 12-1 TSMOM. Score `log(close[t-21]/close[t-252])`. KOSPI top-200 + KOSDAQ top-150 → top-20 동일가중 주간 리밸. **5y bench Sharpe 0.871 / Ann 23.0% / MDD -43% (vs KOSPI 0.656/12.0%)**. bench: `scripts/bench_cs_tsmom_kr.py`.
- `cs_tsmom_crypto_daily.py` — Binance 동일 score. 24h 거래량 top-30 → top-10. **5y bench Sharpe 1.328 / Ann 90.85% / MDD -52% (vs BTC 0.989/51.6%/-77%)**. bench: `scripts/bench_cs_tsmom_crypto.py`.
- `cs_rsi_div_kr.py` — KRX RSI 강세 다이버전스 cross-sectional. `momo_kis_v1` 대체 후보 (5y bench 미실시).
- `cs_bb_macd_kr.py` — KRX BB 하단 회복 + MACD bullish cross. `swing_bb_macd` 대체 (5y bench 미실시).
- `cs_adx_ma_kr.py` — KRX 5/20 EMA 골든크로스 + ADX(14) ≥ 25. `swing_adx_ma` 대체 (5y bench 미실시).
- `cs_rsi_div_crypto.py` — Binance RSI 다이버전스. `momo_btc_v2` 대체 후보.

## 리스크 연동 (필수 — #70 이후)

모든 전략은 일수익률 시계열을 **포트폴리오 리스크 모듈로 공급** 해야 한다. 이 규칙은 선택이 아니라 **구현 계약** 이다.

### 어떻게
```python
from portfolio import StrategyOrchestrator
orch.register_strategy_returns("<strategy_id>", daily_return_series)
orch.refresh_portfolio_risk()
```

- `daily_return_series: pd.Series` — index=날짜, 값=그날 실현 수익률
- `strategy_id` = 전략 파일명 snake_case (예: `momo_btc_v2`)

### 왜
- `src/risk/portfolio.py` 의 LW shrinkage Σ / Meucci ENB / Historical CVaR 는 T×N 수익률 매트릭스를 입력으로 받음 ([[19-portfolio-risk]] §2.2, §3.1, §4.1).
- 전략이 수익률을 공급하지 않으면 `portfolio_risk is None` → 리스크 평가기가 **항상 ALLOW 경로** → 리스크 관리 무력화.
- 본 프로젝트는 **단일 evaluator 경로** (`risk.evaluate`) 유지 — 전략마다 자기 리스크 룰을 재구현하면 `rule_id` 라벨 공간 오염.

## 메타라벨링 훅 추가됨 (#85)

`momo_btc_v2.py` 는 LightGBM 2차 메타라벨러 옵션을 지원한다 (opt-in, 기본 disabled).

### 시그니처 확장 불변식
- `metalabeler=None` 이 기본값 — 기존 호출 경로의 시그니처·동작 변경 없음, 회귀 없음
- `metalabeler_threshold: float = 0.5` — 임계값도 opt-in

### 동작
```python
strategy = MomoBtcV2(metalabeler=fitted_ml, metalabeler_threshold=0.5)
# metalabeler=None (기본) → bypass, 기존 동작 보존
```

- `p_take >= threshold` → 신호 통과, `Signal.win_probability = p_take`
- `p_take < threshold`  → `Signal(action="hold", reason="metalabeler_reject", win_probability=p_take)`

### 전략 시그니처 확장 규칙
새 opt-in 파라미터를 전략 `__init__` 에 추가할 때 **반드시 기본값을 기존 동작과 동일하게** 유지해야 한다. 기존 테스트를 수정하지 않고 통과해야 회귀 없음이 보장된다.

### 프로덕션 구성
메타라벨러 프로덕션 활성화 시 `configs/orchestrator/production.yaml` 참조. 온/오프 이원화 등록 + A/B 모니터링 지표는 `docs/specs/strategies/momo-btc-v2.md` 의 "프로덕션 구성" 섹션 참조.
