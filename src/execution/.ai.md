# src/execution/

## 목적
부모 주문(ParentOrder)을 자식 주문(ChildOrder)으로 분해하는 실행 알고리즘과, KRX 단일가/정지 구간 처리를 담당한다.

## 구조
| 파일 | 역할 |
|------|------|
| `base.py` | `ExecutionAlgorithm` 프로토콜 + 데이터클래스 (Side/TIF/ParentOrder/ChildOrder/Fill/Tick/MarketState/SlippageModel) |
| `market.py` | `MarketAlgo` — 즉시 시장가 1건 |
| `limit.py` | `LimitAlgo` — 단일 지정가 (TIF/post_only) |
| `twap.py` | `TWAPAlgo` — N분할 시간 균등 |
| `vwap.py` | `VWAPAlgo` — 거래량 프로파일 가중 |
| `krx_handler.py` | 단일가/정지 구간 대기 큐 + `SingleAuctionPolicy` |

## 인터페이스 규약
- `plan(parent, state) -> list[ChildOrder]`: 초기 자식 주문 산출
- `on_market_tick(tick) -> list[ChildOrder]`: 틱 도착 시 추가 발주
- `on_fill(fill) -> list[ChildOrder]`: 체결 후 후속 발주
- `cancel()`: 이후 발주 중단

## Phase 1 페이퍼 브로커 (신규)

| 파일 | 역할 |
|------|------|
| `paper_broker.py` | Phase 1 가상 체결 어댑터 — `AsyncBrokerAdapter` Protocol 11개 메서드 완전 구현 |
| `mock_matching.py` | Phase 1 즉시 100% 체결 + 0-슬립 + taker 0.05% 수수료 매칭 엔진 |

### WAL-first 정책
주문 흐름: **WAL submit → matching → 메모리 갱신** 순서 엄수.
WAL 쓰기 실패 시 `kill_switch.trip()` 호출 후 `REJECTED` 반환 (silently swallow 금지).

### Phase 1 결정 사항
- 슬리피지 0, 즉시 100% 체결, partial_fill 미지원
- `cancel_order` no-op (resting order 없음)
- `update_market(state)` — live loop 가 매 tick 호출, `_latest_market_state` 보관
- `from_wal(path, ...)` classmethod — WAL replay 로 PaperBroker 상태 복원

### Phase 2+ 슬리피지 모델 (#109)
- `SquareRootImpact` 클래스 — `SlippageModel` Protocol 구현
  - 공식: `I = sign * k * sigma * mid * sqrt(qty / ADV)`
  - 파라미터: `k` (impact constant, 기본 1.0), `sigma` (변동성, 기본 0.01)
  - `MockMatchingEngine(slippage_model=SquareRootImpact(...))` 으로 주입
  - 기본값 `slippage_model=None` → 0-슬립 유지 (Phase 1 회귀 없음)
  - `MarketState.adv` 가 0 일 때 impact = 0 (ZeroDivision 방지)

### 의존성
- `src/live/wal.py` — `WAL`, `WALWriteFailed`, `replay`
- `src/live/types.py` — `OrderStatus`, `WALEvent`
- `src/ops/kill_switch.py` — `KillSwitch`, `KillSwitchTripped`
- `src/brokers/base.py` — `AsyncBrokerAdapter` Protocol, `OrderRequest`, `OrderAck`, `Position`, `Balance`
- `src/brokers/types.py` — `BrokerFill`

## 규칙
- 모든 알고는 멱등성 보장 (cancel 이후 plan/on_* 호출 시 빈 리스트)
- 슬리피지 모델은 외부 주입 (`SlippageModel` 프로토콜)
- 단일가 핸들러는 알고와 브로커 사이 미들웨어로 배치
