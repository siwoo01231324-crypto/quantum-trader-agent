# src/risk/

## 목적
- 선언적 리스크 룰 DSL — YAML 정책을 파싱·검증하고 스냅샷 단위로 평가 (`dsl.py`).
- 포지션 사이징 수학 — Kelly·Fractional Kelly·Vol Targeting·EWMA σ 의 순수 함수 (`sizing.py`).

## 구조
- `dsl.py` — pydantic 모델(`Policy`, `Snapshot`, `Order`, `Decision`) + `load_policy(path)` + `evaluate(policy, snap)`.
- `sizing.py` — 순수 수학 함수 `kelly_binary`, `kelly_continuous`, `fractional_kelly`, `vol_target`, `ewma_sigma`. I/O·네트워크·LLM 일절 없음. 출력은 `[0, 1]` clamp.
- `__init__.py` — 공개 API.

## 역할
- 모든 주문 발주 직전 `evaluate(...)` 호출 → `Decision` 받기.
- ALLOW가 아닌 모든 결정은 observability로 forward (rule_id 필수).
- 전략의 진입 크기 계산은 `sizing.py` 의 함수를 직접 호출 (예: `MomoBtcV2(sizing_mode="half-kelly")`).
- `sizing.py` 는 `[0, 1]` 범위의 raw size 만 반환; 정책 clamp(`per_position.max_weight_pct` 등) 는 `evaluate(...)` 가 담당.

## 운영 규칙
- 정책 변경은 PR 리뷰 필수.
- 새 룰 추가 시 `tests/test_risk_dsl.py` 에 valid/breach 케이스 동시 추가.
- 사이저 함수 추가 시 `tests/test_risk_sizing.py` 에 수식 레퍼런스·경계값·입력 검증 케이스 동시 추가.
- CLAUDE.md 불변식 #6 — 사이저 결과를 LLM 이 수정하지 않는다.
