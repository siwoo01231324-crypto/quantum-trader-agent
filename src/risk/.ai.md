# src/risk/

## 목적
- 선언적 리스크 룰 DSL — YAML 정책을 파싱·검증하고 스냅샷 단위로 평가 (`dsl.py`, #24 · #70 · #87)
- 포트폴리오 레벨 리스크 지표 — LW shrinkage Σ · Historical CVaR (단일/계층) · Meucci ENB · 평균 ρ (`portfolio.py`, #70 · #87)
- 포지션 사이징 수학 — Kelly · Fractional Kelly · ConsensusKelly · Vol Targeting · UserRiskVolTarget · EWMA σ 순수 함수 (`sizing.py`, #69 · #87)
- 포트폴리오 최적화 사이저 — ERC 볼록 근사 · 2단계 클러스터-HRP (`position_sizer.py`, #87)

## 구조
- `dsl.py` — pydantic 모델(`Policy`, `PerPortfolioRisk`, `Snapshot`, `Order`, `Decision`) + `load_policy(path)` + `evaluate(policy, snap)`. `Snapshot.fear_greed_proxy` (#87 P4), `PerPortfolioRisk.cvar_levels` / `extreme_fear_block` / `extreme_fear_threshold` (#87 P5).
- `portfolio.py` — 순수함수 지표 (LW shrinkage Σ, `historical_cvar`, `historical_cvar_levels` 다중 α #87, Meucci ENB, 평균 ρ) + `compute_portfolio_risk_from_df(...)`. `PortfolioRiskReport.cvar_levels: Optional[dict]` (#87 P5).
- `sizing.py` — 순수 수학 함수 `kelly_binary`, `kelly_continuous`, `fractional_kelly`, `consensus_kelly` (#87 P5), `vol_target`, `user_risk_vol_target` (#87 P1), `ewma_sigma`. I/O·네트워크·LLM 일절 없음. 출력은 `[0, 1]` clamp.
- `position_sizer.py` (#87) — 볼록 ERC (`equal_risk_contribution_convex` — scipy SLSQP, IVP fallback), 2단계 클러스터-HRP (`hrp_with_clustering` — scipy.cluster.hierarchy, k_clusters=None 시 단일 HRP fallback). 모듈 상단에 특허 회피 주석 (포기 특허 US20140081888A1 학술 참고, IBM 양자 컴포넌트 제외).
- `__init__.py` — 공개 API.

## Signal 우선순위 규칙 (#76)
- `Signal.expected_return` 이 있으면 rolling window mu 대신 이 값을 kelly_continuous 입력으로 사용.
- `Signal.win_probability` 이 있으면 kelly_binary 라우팅 (kelly_continuous 아님).
- `Signal.confidence` (0.0 ≠ None): 0.0 은 "계산된 제로", None 은 "미계산" — 라우팅 다름.
- LLM 출력을 expected_return/win_probability/confidence 에 직접 할당 금지 (불변식 #6).

## 역할
- 주문 발주 직전 `evaluate(policy, snap)` → `Decision`. per-order 핫패스.
- 주기 평가기 (일봉/이벤트) 가 `compute_portfolio_risk_from_df(returns_df)` 로 `PortfolioRiskReport` 를 만들어 다음 `Snapshot.portfolio_risk` 로 주입. 이후 동일 `evaluate()` 안에서 `per_portfolio_risk` 블록이 CVaR·상관·ENB 한도를 검사.
- 전략의 진입 크기 계산은 `sizing.py` 의 함수를 직접 호출 (예: `MomoBtcV2(sizing_mode="half-kelly")`).
- `sizing.py` 는 `[0, 1]` 범위의 raw size 만 반환; 정책 clamp(`per_position.max_weight_pct` 등) 는 `evaluate(...)` 가 담당.
- ALLOW가 아닌 모든 결정은 observability 로 forward (rule_id 필수).

## 평가 순서 (first-violation-wins)
`per_trade → per_day → per_portfolio → per_portfolio_risk → per_position → sector_limits → drawdown`

## 기본 action 근거 (PerPortfolioRisk)
- `on_cvar_breach = REDUCE` — CVaR 은 주문 크기에 비례 → 감축이 실제로 손실꼬리 축소
- `on_corr_breach = BLOCK` — 상관은 이미 형성된 포트폴리오 상태 → 신규 주문만 차단
- `on_enb_breach = HALT` — 다변화 구조 자체가 고장 → 사람이 리밸런스

## 운영 규칙
- 정책 변경은 PR 리뷰 필수.
- 새 룰 추가 시 `tests/test_risk_dsl.py` 또는 `tests/test_portfolio_risk.py` 에 valid/breach 케이스 동시 추가.
- 사이저 함수 추가 시 `tests/test_risk_sizing.py` 에 수식 레퍼런스·경계값·입력 검증 케이스 동시 추가.
- `portfolio.py` 는 순수함수만 — LLM 툴 표면에 노출 금지 (불변식 #6).
- `sizing.py` 결과를 LLM 이 수정하지 않는다 (불변식 #6).
- `evaluate()` p99 < 100µs 예산 (`tests/test_portfolio_risk.py::test_evaluate_latency_p99_under_100us`).
