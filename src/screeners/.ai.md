# src/screeners — 룰베이스 종목 스크리닝

## 목적
키움 HTS 조건검색식을 KIS 데이터로 재현하는 evaluator 모듈. 백테스트·라이브 양쪽에서 동일 규칙으로 평가 가능한 순수 함수 인터페이스 제공.

## 구조
- `hts_cond/` — HTS 조건검색식 패키지 (#230 평가 대상)
  - `common.py` — A~G 공통 일간 조건 + profile threshold
  - `dts.py` — 단타 (H=3분봉 20MA 지지)
  - `wait5m.py` — 5분대기작전 (H=상승 정적 VI 근접율 3%)
  - `swing.py` — 스윙 (G까지, 임계값 다름)

## 설계 원칙
1. **순수 함수**: I/O 없음, 입력 dataclass → bool. 백테스트·라이브 모두 같은 코드 경로
2. **룰베이스**: ML/LLM 비의존 (CLAUDE.md 불변식 #6)
3. **profile 분리**: 동일 조건 (A,B,C) 의 임계값만 다른 경우 `ProfileThresholds` 로 주입
4. **재현성**: input dataclass 는 frozen, 결과는 deterministic

## 데이터 소스
- 일간 조건 (A~G): `data/lake/krx_daily/` parquet + KIS `inquire-price` (체결강도)
- 분봉 조건 (단타 H): KIS `inquire-time-itemchartprice` (FHKST03010200)
- VI 조건 (5분대기 H): 전일 종가 × 1.10 계산 (KIS 직접 endpoint 없음, #230 0.2 결정)
