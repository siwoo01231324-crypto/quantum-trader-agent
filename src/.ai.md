# src/

## 목적
프로덕션 코드 루트. 도메인별 서브패키지로 분리된다.

## 구조
| 경로 | 역할 |
|------|------|
| `execution/` | 주문 실행 알고리즘 (TWAP/VWAP/Limit/Market) + KRX 단일가 핸들러 (#25) |
| `observability/` | Prometheus 메트릭 카탈로그 (#26) |
| `ops/` | Kill Switch·자동 트리거 3종·수동 CLI (#27) |
| `tax/` | KR 개인 세법 자동화 (FIFO 매칭·양도세/거래세/배당세·CSV 리포터) (#28) |
| `signals/` | 팩터 라이브러리 (RSI/SMA/ATR/MACD/Bollinger/zscore/donchian) + lookahead-guard + factor registry (#67, #71) |
| `data_lake/` | Binance/KIS OHLCV → Parquet 데이터 레이크 (#67, #106) |
| `backtest/` | 이벤트 루프 백테스트 엔진 + 전략 카탈로그 (momo-btc-v2, momo-kis-v1, breakout-donchian, meanrev-pairs) (#67, #79) |
| `risk/` | 리스크 DSL · 포지션 사이징 (Kelly/Fractional Kelly/Vol Targeting) (#24, #69) |
| `portfolio/` | 포트폴리오 리스크 (LW Σ + CVaR + ENB) + async 오케스트레이터 (#70, #78) |
| `brokers/` | 증권사 API 어댑터 (Binance Futures/KIS) (#68, #73, #106) |
| `live/` | 라이브 실행 프레임워크 (PaperBroker + WAL + WS feed) (#80) |
| `ml/` | AFML 기반 ML (메타라벨링) + Validation 도구체인 (DSR/PBO/CSCV) (#85, #99) |
| `features/` | 이슈-전용 feature 모듈 — Iranyi 8 기법 자동화 (VWMA, EMA projection, multi-TF, time_gate, cross-sectional RS, POC, orderbook flow) (#99) |
| `universe/` | 종목 universe 관리 |

## 규칙
- 외부 I/O는 어댑터 계층에서만 (도메인 모듈은 순수)
- 새 패키지 추가 시 본 파일 표 갱신
