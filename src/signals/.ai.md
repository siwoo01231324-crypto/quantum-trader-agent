# src/signals/

## 목적
무상태(stateless) 팩터·시그널 계산 함수 + 레지스트리 + 룩어헤드 가드 + Parquet 캐시.

## 구조
- `registry.py` — `@register(name, inputs=[...], **defaults)` 데코레이터, `FACTOR_REGISTRY`, `compute()`, `list_factors()`, `DEFAULT_FACTOR_SET = "v1"`. `compute` 는 함수 시그니처에 없는 kwarg를 자동으로 버려 엔진이 OHLCV 전체를 뿌려도 TypeError 가 안 난다. `FactorSpec` 에 `alpha_horizon_bars`, `bar_interval` (closed vocab: 1m/5m/15m/30m/1h/4h/1d/1w), `signal_type` (closed vocab: momentum/mean_reversion/volatility/trend/breakout/event/value/vol/unknown) 추가 — US8433645B1 differs.
- `neutralize.py` — `neutralize(raw, *exposures, method="ols"|"orthogonal")`: OLS 잔차 또는 Gram-Schmidt 직교화로 팩터 노출 제거. 수치 안전장치: cond>1e10 → OLS 폴백. US20140081889A1 abandoned.
- `rsi.py` — Wilder RSI + `detect_divergence` (`shift(1)` 내부 적용). `@register("rsi")`
- `sma.py` — `compute_sma`, `compute_sma_cross` (golden/dead)
- `atr.py` — Wilder ATR (True Range + SMMA)
- `macd.py` — MACD (fast/slow/signal EMA + histogram, `ewm(adjust=False)`)
- `bollinger.py` — Bollinger Bands + `%B` + BandWidth (`ddof=0`)
- `realized_vol.py` — 로그수익률 rolling std × √annualize
- `lookahead_guard.py` — `assert_no_lookahead(...)` append-tail-bar 불변성 검증 (causality 체크; lag-1 강제와는 별개)
- `cache.py` — `to_factor_long` / `write_factor_parquet` / `read_factor_parquet`. `FACTOR_SCHEMA` 장형식 + `partition_path("factor", ...)` 경로. ts 는 UTC 강제.

## 규칙
- **인디케이터 (RSI, SMA, ATR, MACD, Bollinger, RV)**: 인과적(causal) 이면 충분. 내부 `shift(1)` 불필요.
- **결정 신호 (detect_divergence, sma_cross, bollinger-breakout 해석 등)**: 소비 직전에 `shift(1)` 필수 — 룩어헤드 방지.
- **순수 함수**: 외부 상태 의존 없음, 동일 입력 → 동일 출력. `@register` 는 원 함수를 그대로 반환한다.
- **causality 검증**: `lookahead_guard.assert_no_lookahead` 로 append-tail-bar 테스트가 모든 신규 팩터에 자동 parametrize 돼야 한다.

### FactorSpec 필드 (#81)
- `name: str` — 팩터 이름 (registry 키)
- `func: Callable` — 팩터 계산 함수
- `inputs: list[str]` — OHLCV 컬럼명 (예: `["close"]`, `["high","low","close"]`)
- `default_params: dict` — 하이퍼파라미터 기본값 (예: `{"period": 14}`)
- `causal: bool = True` — 바 i 의 출력이 입력 0..i 에만 의존하는지 여부. `False` 인 팩터는 백테스트 엔진의 선계산 경로 진입 시 `AssertionError` (#81). 기본값 `True` — 모든 기본 팩터가 인과적.

### `detect_divergence` 는 팩터 아님
`compute_rsi` 의 출력을 입력으로 받는 **신호 해석기**. 팩터간 DAG 의존성이 레지스트리에 없으므로 등록하지 않는다. 전략이 `context["factors"]["rsi"]` 를 받은 뒤 내부에서 직접 호출한다 (`momo_btc_v2` 참조).
