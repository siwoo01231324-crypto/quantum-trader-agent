# src/brokers/kis/

## 목적
한국투자증권 Open API paper 어댑터 (주 모듈 — KIS가 이 프로젝트의 주 목적).

## 구조
- `auth.py` — OAuth2 토큰 발급 (1분 1회 rate limit, 24h 디스크 캐시, 만료 5분 전 사전 갱신)
- `rest.py` — KISClient (주문·잔고·매수가능 조회)
- `tr_ids.py` — paper/live TR_ID 상수 테이블 (C5: PAPER=VTTC*, LIVE=TTTC*). 읽기 전용 inquiry TR-IDs (paper/live 구분 없음): `TR_ID_BALANCE_SHEET=FHKST66430100`, `TR_ID_INCOME_STATEMENT=FHKST66430200`, `TR_ID_FINANCIAL_RATIO=FHKST66430300` (EPS/BPS/ROE/성장률/마진), `TR_ID_PROFIT_RATIO=FHKST66430400`, `TR_ID_OTHER_MAJOR_RATIOS=FHKST66430500` (EV/EBITDA), `TR_ID_STABILITY_RATIO=FHKST66430600`, `TR_ID_INQUIRE_PRICE=FHKST01010100` (PER/PBR), `TR_ID_DAILY_PRICE=FHKST03010100` (일봉), `TR_ID_INTRADAY_PRICE=FHKST03010200` (분봉, #96).
- `ws.py` — 체결통보 WS (paper: `ws://ops.koreainvestment.com:31000`)
- `crypto.py` — AES-256-CBC + PKCS7 복호화 (key/iv는 구독 응답 body.output에서 추출, Base64)
- `adapter.py` — KISAdapter (BrokerAdapter Protocol 구현)
- `schemas.py` — pydantic v2 응답 모델. 재무 관련: `FinancialRatio` (분기 period-end: eps/bps/sps/roe_val/grs/bsop_prfi_inrt/ntin_inrt/lblt_rate/rsrv_rate), `MarketMultiples` (point-in-time: per/pbr/eps/bps). 시세: `KISDailyBar` (date/open/high/low/close/volume/trade_amt, 문자열→float @field_validator), `KISIntradayBar` (date/time/open/high/low/close/volume/trade_amt, 동일 coercer, FHKST03010200 응답 매핑) (#96).
- `error_map.py` — rt_cd/msg_cd → BrokerError 하위 매핑
- `krx_ticks.py` — KRX 호가단위 테이블 + quantize
- `price_client.py` — KRX 일봉/분봉 OHLCV 수집:
  - `fetch_daily_ohlcv_raw(client, symbol, start, end, period="D") -> list[KISDailyBar]`. TR `FHKST03010100` (`/quotations/inquire-daily-itemchartprice`). 페이지네이션(`CTX_AREA_FK100/NK100`) + rate-limit sleep 0.5s + 429 자체 재시도(Retry-After + 지수 백오프, 최대 3회).
  - `fetch_intraday_ohlcv_raw(client, symbol, target_date, interval="15", *, end_hhmmss="153000") -> list[KISIntradayBar]`. TR `FHKST03010200` (`/quotations/inquire-time-itemchartprice`). 단일일자 페이지네이션(시간 토큰 역방향, 09:00 미만 break + reverse), 0.5s sleep, 동일 429 재시도 헬퍼 패턴. KIS 분봉 API 제약: **당일·최근 30일** (#96).
- `fundamentals_client.py` — 2개 엔드포인트 통합:
  - `fetch_financial_ratio_series(symbol, ...) -> list[FinancialRatio]` (TR `FHKST66430300`, `/finance/financial-ratio`, `FID_DIV_CLS_CODE="1"` 필수, output=분기 list)
  - `fetch_market_multiples(symbol, ...) -> MarketMultiples` (TR `FHKST01010100`, `/quotations/inquire-price`, PER/PBR)
  라이브 검증 완료 2026-04-24 (paper 모의투자). 상세는 `docs/specs/broker-adapter.md §10`.

## 특이사항
- `HANTOO_CREDIT_NUMBER` 파서: regex `^[0-9]{8}-[0-9]{2}$`, 하이픈 split
  - `12345678-01` → `CANO="12345678"`, `ACNT_PRDT_CD="01"`
  - 포맷 위반 시 `ConfigurationError` (명확한 메시지 필수)
- TR_ID: paper=`VTTC*`, live=`TTTC*` — 잘못된 조합 시 `ConfigurationError`
- WS 암호화: AES-256-CBC + PKCS7, 페이로드 `^` 구분자 23 필드
- WS 세션 한도: 종목 41개 + 체결통보 1건
- WS TR_ID: paper=`H0STCNI9`, live=`H0STCNI0`
- KRX 호가단위 quantize 적용 필요 (`krx_ticks.py`)
- Rate limit: 모의 초당 2건, 실전 초당 20건
- 통합 테스트 실행 윈도우: 평일 KST 09:00~15:30 장중만 유효

## Async 레이어 (issue #73, C5/C6a/C6b 완료)

- `async_http.py` — `KISAsyncClient` (httpx.AsyncClient, 3회 retry backoff, `get_token_async` 호출)
- `async_adapter.py` — `KISAsyncAdapter` (`AsyncBrokerAdapter` Protocol 구현, KIS 현물 특이 정책 유지)
  - `aclose` 5단계: ①신규주문 차단 ②WS close ③Binance 전용 step 건너뜀 ④inflight drain ⑤httpx aclose
  - `stream_fills()` → `KISAsyncWebSocket.stream_fills()` AsyncIterator 위임
- `async_ws.py` — `KISAsyncWebSocket` (websockets>=12 기반)
  - `approval_key` WS 구독 직전 1회 발급 (sync ws.py:72-91 패턴 유지, background keepalive 금지)
  - AES-256-CBC 복호화: `crypto.py::decrypt_aes256_cbc_pkcs7` 재사용
  - reconnect backoff: base=1s, cap=10s, jitter=±20%
- `auth.py` (edit) — `asyncio.Lock` + `get_token_async` 추가
  - `_async_lock`: 이벤트 루프 진입 후 지연 초기화 (None → Lock)
  - lazy refresh 구조 유지, background task 절대 금지
  - sync `get_token` 무변경 (기존 테스트 회귀 없음)

### async 금지 항목
- async 파일 내 `time.sleep` / `requests` / `websocket-client` / `threading.Lock` / `httpx.Client`(sync) / `asyncio.run()` 절대 금지
- `approval_key` keepalive background task 도입 금지
