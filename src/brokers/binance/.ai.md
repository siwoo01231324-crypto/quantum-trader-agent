## 목적
Binance USDS-M Futures testnet 어댑터.

## 구조
- `rest.py` — BinanceFuturesClient (HMAC SHA256, recv_window 5s, time drift 보정)
- `ws.py` — user data stream + ReconnectReconciler (C4: 순서 비결정성 대응)
- `adapter.py` — BinanceFuturesAdapter (BrokerAdapter Protocol 구현)
- `schemas.py` — pydantic v2 응답 모델
- `symbol_filters.py` — exchangeInfo 캐시 + quantize (TTL 1h, 런타임 재조회)
- `error_map.py` — -1021/-2010/-2011/-2013/-2019/-4061/-4164 → BrokerError 하위 매핑

## Async 레이어 (issue #73, C3/C4 완료)

- `async_http.py` — `AsyncBinanceFuturesClient` (httpx.AsyncClient, HMAC sign, time drift 보정, `AsyncBinanceRateLimiter` 통합)
- `async_adapter.py` — `AsyncBinanceFuturesAdapter` (`AsyncBrokerAdapter` Protocol 구현)
  - `place_order` 진입 직후 `KillSwitch.assert_allow_order()` + `AsyncTokenBucket.acquire()` 게이트
  - `_closing=True` 후 `place_order` → `BrokerClosedError`
  - `stream_fills()` → `AsyncBinanceUserDataStream.stream_fills()` AsyncIterator 위임
  - `_inflight: list[asyncio.Task]` — inflight REST task tracking (aclose step 4)
  - `aclose` 5단계: ①신규주문 차단 ②WS close ③listenKey keepalive task cancel ④inflight cancel ⑤httpx aclose
- `async_ws.py` — `AsyncBinanceUserDataStream` (websockets>=12)
  - `asyncio.Queue(maxsize=BROKER_FILL_QUEUE_SIZE)` + overflow 정책 {block, drop_oldest, raise}
  - dedup key `(broker_order_id, trade_id)` — fill 중복 제거
  - reconnect: `exponential_backoff` (base=1s, cap=10s, jitter=±20%)
  - Prometheus: `qta_broker_ws_reconnect_total`, `qta_broker_fill_queue_overflow_total`
- `listen_key.py` — `ListenKeyManager` (keepalive 30분, 3회 실패 시 `expiry_event` signal)

### async 금지 항목
- async 파일 내 `time.sleep` / `requests` / `websocket-client` / `threading.Lock` / `httpx.Client`(sync) / `asyncio.run()` 절대 금지
- `uvloop` 의존 금지 (Windows 비호환)
- sync/async 동일 파일 httpx.Client + httpx.AsyncClient 혼재 금지

## 특이사항
- Futures 전용 (Spot 제외 — 후행 이슈)
- STOP/TP 계열 제외 — 후행 Q6
- `position_mode` 기동 시 `ensure_position_mode()` 로 검증 — mismatch 시 BrokerStartupError
- testnet REST: `https://testnet.binancefuture.com` (env `BINANCE_BASE_URL`)
- testnet WS: `wss://fstream.binancefuture.com/ws/<listenKey>` (env `BINANCE_WS_URL`)
- symbol filters 하드코딩 금지 — 매 기동 시 `GET /fapi/v1/exchangeInfo` 재조회, TTL 1h
- quantize_qty: floor-divide 방식 (`(qty // step) * step`) — ROUND_DOWN 보장
- listenKey 수명 60분, 30분마다 keepalive, 24h 사전 재접속
- hedge mode + reduceOnly 조합 → ValidationError 즉시 raise (서버 전송 전)
- time drift: -1021 수신 시 자동 resync + 재시도 1회, 두 번째 -1021은 TimestampError raise
