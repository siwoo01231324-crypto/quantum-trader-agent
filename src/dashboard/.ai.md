# src/dashboard/

## 목적
FastAPI 로컬 대시보드 — localhost:8000 에서 4사분면 HTML UI 와 Prometheus `/metrics` endpoint 제공.

## 구조
| 파일 | 역할 |
|------|------|
| `app.py` | FastAPI 앱 팩토리 `create_app(state)` + standalone `app` 인스턴스. WS `/ws/timeline`, `/strategies`, `/api/run/*`, `/api/account/info` 등 포함 |
| `timeline_broker.py` | in-process pub/sub. drop-oldest back-pressure (#181) |
| `timeline_events.py` | 매매 타임라인 4단계 `event_type` 상수 (#181) |
| `strategy_catalog.py` | `docs/specs/strategies/*.md` frontmatter 로더 (#178). 정규화된 dict list 반환 |
| `run_controller.py` | 거래 시작/정지 컨트롤러 — asyncio task 핸들 + 상태 머신 (#182) |
| `account_info.py` | KIS + Binance 계좌 인증·잔고 조회 + 5초 TTL 캐싱 (#182) |
| `shadow_runs.py` | `logs/shadow/{run_id}/wal.jsonl` 디렉토리 read-only 스캔 — discover/load_detail + 거래소·봉·alive 자동 분류 (#198) |
| `__init__.py` | 패키지 |

## 4사분면 레이아웃
| 사분면 | 내용 |
|--------|------|
| Q1 (좌상) | 손익 — 실시간·일간·월간 PnL 수치 |
| Q2 (우상) | 6종 한도 사용률 게이지 (per_trade·per_day·per_portfolio·per_position·sector·drawdown) |
| Q3 (좌하) | 신호→메타라벨러→주문→체결 타임라인 (최근 50건) |
| Q4 (우하) | 비상정지 4 트리거 상태 + 마지막 발동 시각 + 수동 발동/해제 버튼 |
| Q5 (#182) | 거래 시작/정지 — RunController 기반 [거래 시작]/[거래 정지] 버튼 + 상태 표시 |
| Q6 (#182) | KIS 계좌 (paper) — 계좌번호(마스킹) + 현금/평가금/보유 종목 + 인증 상태 |
| Q7 (#182) | Binance Futures (testnet/live) — API key 마스킹 + USDT 지갑/가용 + 인증 상태 |

## 전략 카탈로그 + 토글 (#178 + #180)

### 데이터 흐름
1. `load_strategy_catalog(specs_dir)` → `docs/specs/strategies/*.md` frontmatter → list[dict].
2. `_enriched_catalog()` 가 각 항목에 `enabled = orchestrator.is_enabled(id)` 머지. orchestrator 미연결 시 모두 `True`.
3. `/strategies` HTML 은 카드 그리드 + 토글 스위치, `/api/strategies` 는 JSON.

### 토글 시 흐름 (D1: 즉시 청산 — 사용자 결정 2026-05-05, CLAUDE.md 불변식 6)
1. 사용자가 카드 OFF 토글 클릭 → JS `confirm()` 다이얼로그 (D2).
2. 확인 시 `POST /api/strategies/{id}/toggle {enabled: false}`.
3. 핸들러는 `state.position_provider(id)` 로 보유 포지션 조회 (callback, 미연결 시 빈 list).
4. `orchestrator.disable_strategy(id, positions=...)` 호출 — `_disabled` set 등록 + WAL `strategy_toggled` 이벤트 emit + 시장가 매도 `OrderIntent` list 반환.
5. 응답에 `liquidation_intents` 포함 — 실제 broker 발주는 caller (live loop) 책임. 본 PR 에서는 **wiring 만**, broker 발주 통합은 후속 이슈.

### DashboardState 신규 필드
- `orchestrator: AsyncStrategyOrchestrator | None` — toggle 핸들러가 enable/disable 호출에 사용. None 이면 503.
- `specs_dir: Path | None` — 기본은 repo root 의 `docs/specs/strategies/`.
- `position_provider: Callable[[str], list[tuple[str, float]]] | None` — 전략 id → 보유 (symbol, qty) 리스트. `scripts/live_run.py` 가 `StrategyPositionStore.get_positions` 로 와이어링 (#192). 토글 OFF 시 caller 가 broker 로 sell intent 발주.

## WebSocket 타임라인 (#181)
- 4 종 `event_type`: `signal_emitted`, `metalabeler_decision`, `order_placed`, `fill_received` (`timeline_events.py`)
- 페이로드 구조 = `WALEvent` JSON dict (`ts`, `event_type`, `schema_version`, `payload`)
- `TimelineBroker` 가입자별 `asyncio.Queue(maxsize=100)` — 가득 차면 oldest drop
- WAL → broker 연결: `WAL(path, observer=broker.publish_walevent_as_dict)` 형식. EXE 런타임 와이어링은 `scripts/live_run.py:_run_pipeline` 가 담당 (#177): `DashboardState(wal_path, timeline_broker)` 구성 → `ShadowConfig.wal_observer = lambda ev: broker.publish(asdict(ev))` 주입 → `_start_dashboard` 가 uvicorn `Server.serve()` 를 메인 asyncio task 안에 띄움
- HTML JS: 연결 끊기면 1초 후 재연결, DOM 100건 cap

## Shadow Runs 뷰어 (#198)

### 데이터 흐름
1. `discover_shadow_runs(state.shadow_log_dir)` → `logs/shadow/*/wal.jsonl` 스캔.
2. 각 run 의 WAL replay → events / corruptions / 거래수 / 마지막 활동 시각 집계.
3. `classify_exchange/timeframe/alive` 으로 거래소·봉·생존 상태 분류.
4. `/shadow_runs` HTML 카드 그리드 + `/api/shadow_runs` JSON.
5. 상세는 `/api/shadow_runs/{run_id}` — `PaperBroker.from_wal` 로 포지션·잔고 복원.

### 분류 규칙
- **거래소**: `BTCUSDT`/`ETHUSDT` 등 USDT 종결 → binance, 6자리 숫자 → kis, 그 외 → unknown
- **봉**: `r4-switch` → 4h, `r6-switch` → 1h, `s2c-voltarget`/`s4-funding` → 4h, `kis`/`krx` 포함 → EOD
- **alive**: 마지막 이벤트가 (봉 단위 × 1.5) 이내 → alive, 봉 × 3 초과 → dead, 그 외 → idle

### Read-only 보장
- WAL 쓰지 않음. 스캔만.
- 디렉토리 부재 / WAL 부재 / corruption → graceful (`status: idle`, `error` 필드)
- broker replay 실패도 graceful — 요약만 표시

### DashboardState 신규 필드
- `shadow_log_dir: Path | None` — `logs/shadow/` 루트. None 이면 `<repo_root>/logs/shadow` 자동 해석.
