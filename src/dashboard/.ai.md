# src/dashboard/

## 목적
FastAPI 로컬 대시보드 — localhost:8000 에서 4사분면 HTML UI 와 Prometheus `/metrics` endpoint 제공.

## 구조
| 파일 | 역할 |
|------|------|
| `app.py` | FastAPI 앱 팩토리 `create_app(state)` + standalone `app` 인스턴스. WS `/ws/timeline` 포함 |
| `timeline_broker.py` | in-process pub/sub. drop-oldest back-pressure (#181) |
| `timeline_events.py` | 매매 타임라인 4단계 `event_type` 상수 (#181) |
| `__init__.py` | 패키지 |

## 4사분면 레이아웃
| 사분면 | 내용 |
|--------|------|
| Q1 (좌상) | 손익 — 실시간·일간·월간 PnL 수치 |
| Q2 (우상) | 6종 한도 사용률 게이지 (per_trade·per_day·per_portfolio·per_position·sector·drawdown) |
| Q3 (좌하) | 신호→메타라벨러→주문→체결 타임라인 (최근 50건) |
| Q4 (우하) | 비상정지 4 트리거 상태 + 마지막 발동 시각 + 수동 발동/해제 버튼 |

## Endpoints
| Method | Path | 설명 |
|--------|------|------|
| GET | `/` | 4사분면 HTML (5초 자동 새로고침, Q3 는 WS 로 incremental) |
| GET | `/metrics` | Prometheus exposition format |
| GET | `/api/pnl` | 손익 JSON |
| GET | `/api/limits` | 한도 사용률 JSON |
| GET | `/api/kill-switch` | 킬스위치 상태 JSON |
| POST | `/api/kill-switch/trigger` | 킬스위치 트리거 발동 |
| POST | `/api/kill-switch/reset` | 킬스위치 트리거 해제 |
| WS | `/ws/timeline?replay=N` | 매매 타임라인 실시간 스트림 (#181). 연결 시 WAL 마지막 N건 replay → `live_ready` 센티넬 → broker fan-out |

## WebSocket 타임라인 (#181)
- 4 종 `event_type`: `signal_emitted`, `metalabeler_decision`, `order_placed`, `fill_received` (`timeline_events.py`)
- 페이로드 구조 = `WALEvent` JSON dict (`ts`, `event_type`, `schema_version`, `payload`)
- `TimelineBroker` 가입자별 `asyncio.Queue(maxsize=100)` — 가득 차면 oldest drop
- WAL → broker 연결: `WAL(path, observer=broker.publish_walevent_as_dict)` 형식. 라이브 운영 와이어링은 `src/runtime/` 부트스트랩에서 수행
- HTML JS: 연결 끊기면 1초 후 재연결, DOM 100건 cap

## 실행

```bash
uvicorn src.dashboard.app:app --host 0.0.0.0 --port 8000 --reload
```
