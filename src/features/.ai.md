# src/features/ — 이랑이 Feature 모듈 (이슈 #99, #185)

## 목적
이랑이 인터뷰 ([[iranyi-vwma-2026-04-27]]) 의 8 기법을 자동화 가능한 결정론적 feature 함수로 변환한다. 사전 등록 factorial 실험 (Variant A~H) 의 입력. Bench script `scripts/bench_iranyi_variants.py` 가 직접 호출.

`src/signals/` (production signal 라이브러리) 와 분리: 본 디렉토리는 **이슈 #99 실험 전용** 이며, factor registry 등록은 하지 않는다 (winning variant 가 정해지면 `src/signals/` 로 promote — 후속 작업).

## 의존 (Inter-module)
- `multi_tf.py` → `vwma.py` (`vwma()` 사용)
- `ma_projection.py` 내부 EMA 계산 (의존 없음)
- 기타 모듈은 서로 독립 — 단방향 의존 그래프 (순환 없음)

## Lookahead Guard 사용 규약
- 본 디렉토리의 어떤 모듈도 `assert_no_lookahead` 의 자체 구현을 두지 않는다.
- 단위 테스트 (`tests/test_iranyi_features.py`) 는 `from src.signals.lookahead_guard import assert_no_lookahead` 로 재사용한다.
- 인과성 패턴:
  - `shift(1)` 또는 `shift(2)` 사용 — `vwma_cross` 가 t-1 cross 를 t 에 반영
  - `rolling(window).apply(...)` — 자연스럽게 t 시점 이후 배제
  - `resample(label='right', closed='right')` — 멀티프레임 alignment 의 핵심 (multi_tf, orderbook_flow)
- `time_gate` 는 OHLCV 입력이 아니므로 `assert_no_lookahead` 의 표준 (`inputs=["close",...]`) 와 시그니처 다름 — 별도 인과 테스트 사용.

## Gotcha
- `vwma_cross` 는 `shift(1)` 두 번 적용 → 첫 2 bar 는 None
- `multi_tf_alignment` 는 HTF bar 의 close timestamp 가 발생 시점이 아님 — 한 HTF step 만큼 추가 shift 가 필수 (코드 내 처리됨)
- POC 는 brute-force histogram → 시간복잡도 O(T·n_bins). T=수백만 bar 시 cython/numba 최적화 후순위.
- `time_gate` 의 timezone 변환: naive index 는 UTC localize 후 변환 — 입력 인덱스가 UTC 가정인 경우만 정확함
- UBAI 의 universe 변경 시점 (월 1일) — point-in-time 정확성 보장 필요 (lookahead 없음, fetcher 가 r_date 시점 데이터만 반환해야 함)
