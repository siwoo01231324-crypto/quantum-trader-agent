# src/ml/pipelines/

## 목적
메타라벨러 학습/평가 파이프라인 — 자산군 별 entry-point 함수.

## 구조
- `kis_cross_validation.py` — KRX (`momo-kis-v1`) 메타라벨러 학습 + walk-forward 평가 파이프라인 (#97).

## 규칙
- 핵심 로직은 `src/ml/retrain_pipeline.py` 의 `train_and_save()` 호출로 유지 — 자산군 차이는 파라미터 (`costs_bps`, `holding_bars`, `periods_per_year`) 로만 표현.
- `run_kis_pipeline_pooled` 가 저장하는 `manifest.json` 의 `training_window` 는 BTC `train_and_save` 와 동일하게 `start`/`end`/`n_events` 3 필드 필수 — `scripts/cross_asset_compare.py` 가 `data_window` 컬럼을 채우는 데 사용. 누락 시 placeholder fallback (#155).
