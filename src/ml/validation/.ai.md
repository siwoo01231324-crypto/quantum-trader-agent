# src/ml/validation/ — AFML Multi-Testing Correction Toolkit

## 목적
다중 가설 검정 (multi-testing) 보정을 위한 통계 도구. `src/ml/cv.py::PurgedKFold` 와 함께 AFML (López de Prado 2018) 기반 검증 SOP (`docs/background/12-validation-protocol.md` §3.7) 를 구현한다.

본 이슈 #99 의 8 variant factorial 실험에서 사용 (Bench script `scripts/bench_iranyi_variants.py`). 향후 다른 전략에서도 재사용 가능한 프로젝트 인프라.

## 파일 구성
| 파일 | 역할 |

## 프로젝트 게이트 (`docs/background/12-validation-protocol.md` §3.7)
- **DSR ≥ 0.95**: 다중 시행 보정 후 통계적 유의 (5% 수준)
- **PBO ≤ 0.2**: 백테스트 과적합 확률 한계
- 추가: OOS MDD < 25%, monthly hit rate ≥ 50% (5년+ 구간)

## PSR 공식 (Bailey & López de Prado 2014, Eq.4)
```
se_sr = sqrt((1 - skew*SR + (kurt-1)/4 * SR^2) / (n-1))
PSR   = Phi((SR - SR_bench) / se_sr)
```

## DSR 공식 (Theorem 2)
```
SR0 = sqrt(Var(SR_estimates)) * ((1-γ)*Z⁻¹(1-1/N) + γ*Z⁻¹(1-1/(N*e)))
γ   = Euler-Mascheroni ≈ 0.5772156649
DSR = PSR with benchmark = SR0
```

## CSCV 알고리즘 (n_groups=16 → C(16,8)=12_870 조합)
1. T 시점을 `n_groups` 등분 contiguous 블록
2. C(n_groups, n_groups//2) 조합 각각:
   a. IS = 절반 블록, OOS = 나머지
   b. IS-best variant (mean return) 선정
   c. OOS rank 의 relative position λ = rank / N
3. PBO = fraction(λ > 0.5)

## `returns_matrix` 입력 명세 (`scripts/bench_iranyi_variants.py` 와 합의)
- Shape: (T, N) — T = OOS daily return concat (PurgedKFold test split chronological), N = variant 수
- 거래 없는 날: 0.0
- 정렬: chronological, 셔플 금지

## Gotcha
- `var_sr <= 0` 엣지 케이스 (모든 trial 동일 SR) → PSR with benchmark=0 fallback
- `n_obs < 30` 시 PSR 불안정 경고
- C(16,8) brute-force = 12870 조합 — variant 수준 (N) 이 작아 trivial. 단, n_groups 키우면 조합 폭발 (C(20,10)=184_756)
- `_rankdata_descending` 은 ties 평균 처리 — `scipy.stats.rankdata(method='average')` 와 호환
