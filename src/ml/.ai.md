# src/ml/ — 메타라벨링 ML 모듈

## 목적
2차 메타라벨링 필터 구현. 규칙 기반 전략 신호의 false positive를 LightGBM으로 쳐낸다.
스펙: `docs/specs/ml/meta-labeling.md` | 이론: `docs/background/35-meta-labeling-lopez-de-prado.md`

## 파일 구성
| 파일 | 역할 |
|------|------|
| `labeling.py` | Triple-barrier 라벨링 (López de Prado Ch.3) — `triple_barrier_label()` |
| `cv.py` | Purged K-fold + embargo CV (Ch.7) — `PurgedKFold` |
| `meta_labeler.py` | LightGBM 2차 필터 — `MetaLabeler`, `MetaLabelerConfig` |
| `walkforward.py` | Expanding / rolling walk-forward 분할기 — `WalkForwardSplitter`, `WalkForwardConfig` |
| `drift_detector.py` | 모델 드리프트 감지 — `compare(prev, new, bench) -> DriftReport` (accuracy_delta ≥ 0.05 OR sharpe_delta ≥ 0.3 시 triggered) |
| `retrain_pipeline.py` | 재학습 공개 API — `load_ohlcv_from_lake`, `build_events_and_features`, `label_events`, `run_cv`, `run_cv_extended`, `train_and_save` |
| `scoring.py` | ML 평가 지표 — `annualized_sharpe`, `deflated_sharpe_ratio`, `max_drawdown`, `pr_auc_score`, `sharpe_improvement_ratio` (#97) |
| `pipelines/` | 자산군별 학습 파이프라인 entry-point — `kis_cross_validation.run_kis_pipeline` (#97) |
| `reporting/` | 교차 자산 비교 리포팅 — `cross_asset_compare.build_comparison_table`, `judge_hypothesis`, `render_markdown` (#97) |

## 공개 API (`src/ml/__init__.py` 경유)
```python
from src.ml import (
    triple_barrier_label,   # labeling.py
    PurgedKFold,            # cv.py
    MetaLabeler,            # meta_labeler.py
    MetaLabelerConfig,      # meta_labeler.py
    WalkForwardSplitter,    # walkforward.py
    WalkForwardConfig,      # walkforward.py

```

## 호출 경로
전략 `on_bar` → `MetaLabeler.win_probability(feat)` → `Signal.win_probability` → Position Sizer (#69)

## 볼트 사용 안 함
순수 수치 계산 모듈. Obsidian MCP 서버 연결 불필요.
CLAUDE.md "새 에이전트 추가 시 볼트 연결 필수" 예외 근거:
이 모듈은 에이전트가 아닌 라이브러리 함수이며, 시계열 수치 데이터만 처리하므로 문서 볼트와 무관하다.

## 불변식
- `deterministic=True`, `force_col_wise=True`, `random_state=42` — 재현성 강제 (CLAUDE.md #6)
- `t_touch > entry_ts` 엄격 부등식 — 룩어헤드 금지 (#71 가드 동일 경계)
- `costs_bps` 거래비용 라벨 반영 필수 (train-live Sharpe 괴리 방지)
- LLM 개입 금지 (CLAUDE.md #6) — `predict_proba` → `win_probability` 직접 매핑

## Gotcha
- purged CV timestamp 경계: `t1`(라벨 확정 시점) 기준으로 train/test 겹침 판단
- Windows 환경 LightGBM 재현성: `force_col_wise=True` 필수. GPU 없으므로 CPU 경로 고정
- `MetaLabeler.win_probability` 는 후처리(캘리브레이션) 없음 — 별도 이슈로 분리
- 모델 아티팩트(`models/`) 는 `.gitignore` 적용 — 커밋 금지
