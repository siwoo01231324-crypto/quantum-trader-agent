# src/portfolio/

## 목적
복수 전략 출력을 **한 곳에서 조립** 해 포트폴리오 리스크 모듈 (`src/risk/`) 로 흘려보내고, 주문 의도를 정책과 최신 리스크 리포트에 대해 게이팅하는 레이어.

## 구조 (#78 완료 후, #94 확장)

2-class composition 구조 (D1):
- `orchestrator.py` — `_SyncStrategyOrchestrator` (private, lock-free sync, #70 stub rename). 하위 호환 alias: `StrategyOrchestrator = _SyncStrategyOrchestrator`. `compute_fear_greed_proxy(price_history, window=252)` 헬퍼 (#87 P4) — 가격 기반 `close_latest / rolling_max(window)` 정규화, 소셜 크롤 없음.
- `_async_orchestrator.py` — `AsyncStrategyOrchestrator` (public async, #78). `_SyncStrategyOrchestrator` 를 composition 으로 감쌈.
- `config_loader.py` — `load_orchestrator_from_yaml(path, policy) -> AsyncStrategyOrchestrator` (#94). YAML 파싱 → importlib 동적 클래스 로드 → `metalabeler.load_path` 치환 → 전략 등록. strategy_id 중복 시 ValueError. MetaLabeler.load 실패 시 RuntimeError (fail-fast).
- `_strategy_adapter.py` — `_StrategyAdapter` (#94). sync `on_bar(bar, history, context)` → 오케스트레이터 `on_bar(ctx)` 어댑터. `MomoBtcV2` 등 sync Protocol 전략을 AsyncStrategyOrchestrator 에 연결.
- `order_intent.py` — `OrderIntent` dataclass (frozen=True, slots=True). `run_bar` 반환 타입.
- `sizing.py` — `resolve_size(signal, recent_returns)`. Signal 메타데이터 → Kelly 분기. `src/risk/sizing.py` (Kelly 수식) 와 다른 개념.
- `__init__.py` — 공개 API: `AsyncStrategyOrchestrator`, `OrderIntent`. `_SyncStrategyOrchestrator` 는 private.

## 역할
- 전략 N개의 일수익률 → T×N DataFrame → `risk.compute_portfolio_risk_from_df`
- 생성된 `PortfolioRiskReport` 를 `Snapshot.portfolio_risk` 로 주입
- 가격 히스토리 → `compute_fear_greed_proxy` → `Snapshot.fear_greed_proxy` 로 주입 (`evaluate_order()` 에서 조립, `refresh_portfolio_risk()` 시그니처 불변)
- `risk.evaluate(policy, snap)` 한 번 호출로 per-order 판정 + per_portfolio_risk 판정을 **같은 precedence chain** 에서 처리 (`extreme_fear_block` 평가 시 `snap.fear_greed_proxy` 직접 참조)

## 현재 스코프 (#70, #76 에서 제공)
- Sync API: `register_strategy_returns`, `refresh_portfolio_risk`, `evaluate_order`
- `strategy_reliability_score(strategy_id)` → [0,1]: `convex_base × drawdown_gate`. NaN 가드: T<20 → 0.0. KR101139626B1 differs: 곱셈 게이트(불연속 지시함수) ≠ KR 가산합.
- **reliability metric**: drawdown_gate = clip(1 − max_dd/0.20, 0, 1). max_dd ≥ 20% → 하드 제로.
- 2+ 전략이 있어야 report 생성. 1전략 이하 / T<2 → `ALLOW` 경로 유지 (회귀 zero)

## 완료 (#78)

### run_bar (async tick driver)
- `AsyncStrategyOrchestrator.run_bar(ts, market_snapshot)` — 등록된 모든 전략을 `asyncio.gather` 로 동시 실행, 각 전략 출력에 `resolve_size` + `risk.evaluate` 를 적용하고 ALLOW 된 것만 `list[OrderIntent]` 로 반환.
- 동기 전략(`on_bar` 는 일반 함수): `asyncio.to_thread` 로 스레드풀에서 실행.
- 비동기 전략(`on_bar` 는 코루틴): `asyncio.create_task` 로 직접 실행.
- 전략 구분은 `inspect.iscoroutinefunction(strategy.on_bar)` 로 dispatch.

### 격리 / 쿼런틴 (circuit-breaker)
- 전략이 예외(RuntimeError, asyncio.TimeoutError 등) 를 3회 연속 발생시키면 `_quarantined` set 에 추가 → 이후 `run_bar` 에서 skip.
- 성공 시 해당 전략의 실패 카운터(`_fail_count`) 를 0으로 reset → 쿼런틴 조건 재시작.
- `quarantined_strategies` property 로 현재 쿼런틴 목록 조회 가능 (frozenset).

### 백그라운드 리프레시 (bar-clock / wall-clock)
- **bar-clock 모드**: `AsyncStrategyOrchestrator(policy, refresh_every_n_bars=N)` — `run_bar` 가 N번 호출될 때마다 `refresh_portfolio_risk_async` 자동 실행.
- **wall-clock 모드**: `start_risk_refresh_loop(interval_sec, jitter_frac)` / `stop_risk_refresh_loop()` — 백그라운드 asyncio Task 로 주기적 리프레시. 지터(±jitter_frac) 포함.
- `refresh_portfolio_risk_async` 는 `asyncio.Lock` 으로 동시 쓰기 보호.

### BrokerExecutor (DEFERRED to #80)
- `run_bar` 반환값 `list[OrderIntent]` 를 소비하는 레이어. #80 에서 구현 예정.

### Live-scanner stop/TP 매니저 (#227 S2)

- `live_position_risk.py::LivePositionRiskManager` — 매 tick `evaluate(symbol, last_price, ts)` 호출 시 보유 (strategy_id, symbol) 페어를 `StrategyPositionStore` 에서 조회 + 평균매수가를 `PnLAggregator._cost_basis` 에서 조회 → stop/TP/trailing 임계값 breach 시 SELL `OrderIntent` emit + WAL `position_stop_triggered` 기록.
- 정책 등록: `register_strategy_policy(strategy_id, stop_loss_pct, take_profit_pct, trailing_stop_pct=None)`. live_run.py 가 LiveScannerMixin 전략의 class attr 을 읽어 wire-up (S3 예정).
- 청산 우선순위: `stop_loss` → `take_profit` → `trailing_stop` (trailing 은 high_water > entry 일 때만 작동, stop_loss 와 중복 방지).
- 모든 산술 `Decimal` — `PnLAggregator` 정합성. high_water `dict[(sid, symbol), Decimal]` 는 보유 0 도달 시 자동 클린.

### Live-scanner per-symbol dispatch (#227 S1)

- `run_bar` 의 strategy dispatch 분기에서 `getattr(strategy, "is_live_scanner", False)` 가 True 이고 `market_snapshot["ohlcv_history"]: dict[symbol, DataFrame]` 가 비어있지 않으면 universe 의 각 symbol 마다 별 task 를 spawn — 각 task 는 단일 symbol 의 `market_snapshot = {symbol, history, price, equity_krw}` 와 `factors = universe_factors.get(symbol, {})` 를 받음.
- legacy 단일 dispatch path (cs_*, momo_*) 는 변경 없음 — `is_live_scanner` 미선언 + `ohlcv_history` 부재 시 동일 ctx 로 단일 호출.
- per-symbol 결과 OrderIntent 는 `task_symbols` parallel list 로 추적 → 적합한 symbol 로 발행. legacy 는 `task_symbols[i] is None` → `market_snapshot["symbol"]` fallback.
- **격리 (quarantine) dedup**: live-scanner 가 N 종목 모두에서 throw 시 `_fail_count` 가 1만 증가 (consecutive-bad-tick 의미 보존). reset 도 같은 tick 에 sibling task 가 throw 했으면 보류.

### Runtime ON/OFF 토글 (#180)
- `enable_strategy(id)` / `disable_strategy(id, *, positions=None)` — runtime 활성/비활성 전환. `_disabled: set[str]` 관리.
- `is_enabled(id) -> bool`, `disabled_strategies` (frozenset) property.
- `run_bar` 가 `_disabled` 항목을 skip — 신규 시그널 차단.
- **즉시 청산 (D1)**: `disable_strategy(id, positions=[(symbol, qty), ...])` 가 각 포지션에 대해 `OrderIntent(side='sell', reason='strategy_disabled_liquidation')` list 반환. caller 가 broker 로 발주.
- **WAL 감사 로그**: 생성자 `wal_observer: Callable[[WALEvent], None] | None` 받음. 활성 전이 시 `WALEvent(event_type='strategy_toggled', payload={strategy_id, enabled, actor='user'})` 호출. observer 예외는 swallow + log warn.
- **Idempotent**: 이미 disable 인 전략을 재차 disable → state 유지, WAL 재발행 X. positions 가 주어지면 청산 intents 는 매번 새로 생성 (caller 가 최신 보유분을 주입한다고 가정).
