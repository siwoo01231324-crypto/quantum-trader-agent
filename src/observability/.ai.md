# src/observability/

## 목적
Prometheus 메트릭 23종을 단일 객체(`Metrics`)로 노출. 스펙은 `docs/specs/observability.md`.

## 구조
| 파일 | 역할 |
|------|------|
| `metrics.py` | `Metrics` 클래스 + 23종 카탈로그 + `get_registry()` 싱글톤 |
| `alerts.py` | 알림 어댑터 — `notify(level, title, body, fields)` → Slack webhook / Telegram Bot / stdout 분기 (fail-soft) |

## 규칙
- 모든 메트릭은 `qta_` 접두사
- 라벨은 저카디널리티 (symbol은 화이트리스트만)
- 새 메트릭 추가 시 `METRIC_NAMES`와 본 파일 동시 갱신
- 라벨 없는 메트릭에 라벨 추가 금지 (기존 시그니처 불변)
