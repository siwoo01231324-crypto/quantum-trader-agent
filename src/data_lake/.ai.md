# src/data_lake/

## 목적
프로젝트 데이터 레이크의 스키마 정의와 기본 검증 헬퍼.

## 구조
- `schema.py` — 모든 테이블(OHLCV/Orderbook/Trade/Factor/Meta/Fundamentals) 스키마와 `validate_schema`, `partition_path` 헬퍼.
- `__init__.py` — 공개 API 재노출.
- `fetcher.py` — Binance REST klines → OHLCV DataFrame → Hive-partitioned Parquet. 1000캔들/요청 페이지네이션, 429 지수백오프 재시도(최대 3회), 0.5s 요청 간격. 진입점: `fetch_binance_klines()`, `save_ohlcv_parquet()`. KIS 일봉: `fetch_kis_daily_ohlcv(symbol, start, end, *, auth, app_key, app_secret, cano, acnt_prdt_cd, paper=True) -> pd.DataFrame` — OHLCV_SCHEMA 정규화(source="kis", freq="1d"), 내부에서 `price_client.fetch_daily_ohlcv_raw` 호출. KIS 분봉: `fetch_kis_intraday_ohlcv(symbol, start, end, interval="15", *, auth, ...) -> pd.DataFrame` — OHLCV_SCHEMA 정규화(source="kis", freq=f"{interval}m"), 일자별 loop + 0.5s sleep + KRX 영업일 + 휴일 skip + KIS 30일 boundary warning(#96).
- `fundamentals_store.py` — `to_fundamentals_frame(FinancialRatio | MarketMultiples | list[...])` → FUNDAMENTALS_PIT_SCHEMA DataFrame. FinancialRatio: fiscal_date "YYYYMM" → period_end (월말 Asia/Seoul), source="kis_fin_ratio_v1", fiscal_period="YYYYMM". MarketMultiples: period_end=ingested_at UTC, source="kis_market_mult_v1", fiscal_period="pit". 단위: eps/bps/sps=krw, roe/growth/margin/debt=pct, per/pbr=ratio. 소비자: `src/brokers/kis/fundamentals_client.py`.

## 역할
- 적재/조회 코드는 반드시 여기 정의된 스키마를 import해서 사용.
- 컬럼 추가 시 `docs/specs/data-lake-schema.md` 의 운영 규칙(섹션 7) 준수.
- `src/signals/cache.py` (#71) 가 `FACTOR_SCHEMA` 와 `partition_path("factor", factor_set=..., ts_year=..., ts_month=..., symbol=...)` 를 소비해 팩터 결과를 장형식 Parquet 으로 적재한다.
- `FUNDAMENTALS_PIT_SCHEMA` (9컬럼: symbol/announce_date/period_end/fiscal_period/metric/value/unit/source/ingested_at) — KIS 재무비율 PIT 데이터 저장 스키마. 타임존 규칙: period_end/announce_date = Asia/Seoul, ingested_at = UTC.
