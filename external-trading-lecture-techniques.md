## 0. 전체 프레임 — MTR 3종 매매법

| 약자 | 풀네임 | 시간축 | 한 줄 정의 |
|---|---|---|---|
| **M** = MG | Momentum Gap | 단타 (15분 / 4시간) | 볼린저밴드 평균회귀 단타 |
| **T** = TF | TFCM (이름 추정) | 스윙 (일/주) | 일목구름표·피보나치·다이버전스 조합 스윙 |
| **R** = RS | (의미 불분명) | 가변 | 알트코인 "세력매매" — 온체인·펀딩비·언락 활용 |

강의에서 실제로 깊이 다룬 것은 **MG와 캔들/타임프레임 기초**가 거의 전부다. TF·RS는 정규강의 콘텐츠라며 자세한 설명 보류.

## 1. MG — 볼린저밴드 평균회귀 단타

### 1.1 핵심 가설

- 정규분포 가정 하에 볼린저밴드(20·2σ) **안에 머무는 확률 95.4%**, 밖으로 이탈하는 확률 4.6%.
- 따라서 2σ 이탈은 "고무줄" 비유처럼 중앙선(20기간 이동평균)으로 회귀할 확률이 통계적으로 우세.
- **상단 이탈 → 쇼트, 하단 이탈 → 롱**.

### 1.2 진입 규칙

진입 방법 2가지 (둘 다 "이탈 직후 즉시 진입 금지"가 공통):

**방법 A — 캔들 패턴 트리거**
1. 4시간봉 볼린저밴드 상단/하단 **이탈 확인 (종가 기준)**.
2. 곧장 들어가지 말고 "대기 모드".
3. 15분봉 또는 5분봉으로 내려와서 **반전 캔들 패턴**(엥걸핀, 도지, 해머, 유성형 등) 발생 + 거래량 동반 확인.
4. 캔들 확정 후 진입.

**방법 B — 추세선 돌파 더블 확인**
1. 4시간봉에서 추세선 작도.
2. 추세선 돌파 확인.
3. 낮은 타임프레임(15분/5분)에서 **돌파 재확인** (눌림 후 재돌파 또는 리테스트).
4. 진입.

### 1.3 청산 규칙

**익절 (분할)**
- 1차 익절: 15분봉 **반대편 볼린저밴드 도달 시 80% 청산**.
- 2차 익절: 잔량 20%는 **4시간봉 중앙선(미들, 21MA)** 도달 시 청산.
- 결과 손익비(R) 약 **1 : 2.5**.

**손절**
- 방법 A로 진입했으면 → **트리거가 된 반전 캔들이 무효화(돌파/이탈) 되는 순간 즉시 컷**.
- 방법 B로 진입했으면 → **추세선 재이탈 시 즉시 컷**.

### 1.4 밴드라이딩 함정 회피

- 강한 추세장에서는 가격이 밴드를 따라 계속 미끄러지는 "밴드라이딩" 발생 → 평균회귀 가설이 깨짐.
- 회피책: 이탈 직후 진입 금지, **반전 캔들이 거래량과 함께 축소될 때까지 대기**.

## 2. 캔들 패턴 — 추세 끝/지지·저항에서만 유효

강의의 핵심 메시지: **"캔들은 외우는 게 아니라 매수·매도 심리의 결과로 이해하라."**

### 2.1 다루는 패턴

| 패턴 | 형태 | 의미 |
|---|---|---|
| 도지 (Doji) | 시가≈종가, 위·아래꼬리 | 매수·매도 교착, 추세 전환 시그널 |
| 해머 | 짧은 몸통 + 긴 아래꼬리 | 하락 끝 반전 (매도가 매수에 압도당함) |
| 유성형 (역해머) | 짧은 몸통 + 긴 위꼬리 | 상승 끝 반전 (매수가 매도에 압도당함) |
| 장악형 (Engulfing, "엥걸핀") | 직전 캔들 몸통을 완전 포함하는 반대색 큰 캔들 | 강한 반전 신호 |
| 먹구름형 / 관통형 | 부분 침투형 장악형 | 약한 반전 신호 |

### 2.2 유효성 조건

- ✅ **추세 끝, 또는 지지·저항 구간**에서 발생해야 신뢰도↑.
- ❌ 횡보 한가운데, 추세 중간에서는 **트랩** 가능성↑ → 무시.
- ✅ **거래량 동반** 시 신뢰도↑.
- ✅ 직전·직후 캔들과의 관계(컨텍스트)가 패턴 자체보다 더 중요.

## 3. 멀티 타임프레임 탑다운

### 3.1 분석 순서

```
월봉 → 주봉 → 일봉 → 4시간봉 → 15분봉
```

### 3.2 원칙

- **큰 프레임 추세가 작은 프레임을 이긴다** — 일봉 상승 추세 중 15분봉 단기 하락은 매수 기회로 해석.
- **큰 프레임 캔들 패턴이 더 신뢰성 높음** — 월봉/주봉 도지·반전 캔들 > 일봉.
- 데이트레이딩 주 타임프레임은 **15분봉**, 1시간/4시간 병행.
- 진입 시그널은 작은 프레임에서, 방향은 큰 프레임에서.

---

## 4. 근거 중첩(스택) 규칙

**"단일 지표·단일 시그널로 매매하지 말 것."**

- 근거 1개 → 승률 50%
- 근거 2개 중첩 → 승률 ~70% (강사 주장, 검증 불가)
- 근거 3개+ → A급 셋업

근거의 예: 볼린저 이탈 + 반전 캔들 + 추세선 + 거래량 + 다이버전스 + 더 큰 타임프레임 지지/저항 등.

### 5.1 셋업 등급별 사이징

| 등급 | 근거 수 | 사이즈 | 익절 목표 |
|---|---|---|---|
| **A급** | 다수 중첩 | 크게 ("박스 베팅") | 길게 |

| **C급** | 2개 (최소) | 작게 | 짧게 |

### 5.2 손절 / 데일리 정지

- **연속 3회 손절 시 당일 매매 중단** ("타임폐") — 멘탈/스피럴 방지.

### 5.3 종목 선정

- 데이트레이딩은 **유동성·변동성 풍부한 종목만** (슬리피지 회피).
- 비주류·저유동성 알트 단타 금지.

## 8. 비판적 검토 (본 프로젝트 기준)

이 강의 기법들을 그대로 받아들이면 안 되는 이유와, **이 프로젝트에 차용하려면 무엇을 추가로 검증해야 하는가**:

### 8.1 통계적 결함

- "볼린저밴드 안 95.4% / 밖 4.6%"는 가격 수익률이 **정규분포라는 가정** 위에서만 성립. 실제 암호화폐 수익률은 **두꺼운 꼬리(fat-tail)** 분포 → 2σ 이탈 빈도가 4.6%보다 훨씬 잦고, **이탈 후 회귀 확률도 가정보다 낮음** (밴드라이딩 = 추세장).
- 강의는 "밴드라이딩 주의"로 우회하나, 이는 사후적 변명이며 사전 필터링 규칙(예: ADX > X, 변동성 레짐 분류)이 없다.

### 8.2 검증 부재

- 강의는 "승률 75.1%" 같은 수치를 클로드 코드 백테스트라며 제시하지만 **데이터 기간·심볼 유니버스·수수료·슬리피지·다중검정 보정(PBO)이 모두 비공개**.
- 본 프로젝트 규칙: **5y backtest + Deflated Sharpe + PurgedKFold + PBO 통과 못 하면 production 진입 금지**. 이 강의 기법을 차용하려면 동일 검증 필수.

### 8.3 차용 가능 부분 (우리 프레임에 정직하게 매핑)

| 강의 항목 | 우리 프로젝트 매핑 | 차용 우선순위 |
|---|---|---|
| 볼린저 평균회귀 단타 (MG) | `live_bb_lower_bounce.py` 와 거의 동일 컨셉 — 이미 구현됨 | 낮음 (중복) |
| 캔들 패턴 반전 + 거래량 | `src/signals/` 에 새 시그널로 추가 후 백테스트 | 중간 |
| 추세선 돌파 더블 확인 | 멀티 타임프레임 confirmation rule — 시그널 stacking 로직 | 중간 |
| 멀티 TF 탑다운 | 이미 일부 적용. 시그널 stacking 시 명시적 cross-TF 규칙화 | 중간 |
| 근거 중첩 → 사이징 차등 (A/B/C급) | `src/risk/sizing.py` `consensus_kelly` 와 컨셉 동일 (이미 구현, `#87`) | **이미 구현됨** |
| 연속 3회 손절 시 정지 | `src/risk/dsl.py` per_day max_loss + drawdown 게이트로 표현 | 낮음 (이미 있음) |

### 8.4 차용 금지 부분

- **고배율 선물 + 풀시드 베팅 발언** — 강사 본인이 청산·핵킹으로 30~70억 손실을 인정. 사이즈 정책(불변식 + risk DSL `max_leverage: 1.0`)에 반함.
- **"감으로 들어가지 말고 시스템으로"**라는 슬로건은 동의하나, 강의가 제시한 시스템 자체가 비검증이므로 슬로건만 차용하고 시스템은 본 프로젝트의 AFML 파이프라인으로 대체.
- **에어본/찐다 지표 등 외부 인디케이터** — 코드 미공개. 차용하려면 같은 시그널을 직접 구현하고 `signals/registry.py` 에 등록 + causal 검증.

## 9. 결론

이 강의의 기법 자체는 **시중 무료 자료 수준의 기초 기술적 분석(볼린저 평균회귀 + 캔들 + 멀티 TF + 근거 중첩 + 손절 규칙)**이며, 본 프로젝트는 이 중 상당 부분을 이미 더 엄밀한 형태로 구현(live-scanner 패러다임, `consensus_kelly`, risk DSL)하고 있다.

차용 가치가 있는 부분은:
1. **캔들 패턴 신호를 `src/signals/`에 명시적 추가** (현재 없음) — 단, 추세 끝/지지·저항 컨텍스트 필터 포함.
2. **추세선 작도 + 더블 confirmation 규칙**의 시그널화.
3. **셋업 등급(A/B/C) → 사이즈 차등화**를 더 명시적으로 (이미 부분 구현된 consensus_kelly의 단계 확장).

위 3가지는 별도 spec 작성 → `docs/specs/strategies/` 후 5y 백테스트 → DSR/PBO 통과 시 channel-in 검토.

## 10. 실제 채택 진행 상황 (2026-05-20)

위 1번(캔들 패턴 게이트)을 가장 lightweight 방향으로 시험관 구현 후 1y BTC+ETH sweep 으로 검증.

- **전략**: `live_mg_bb_reversal` (live-scanner 패러다임)
- **코드**: `src/backtest/strategies/live_mg_bb_reversal.py`
- **스펙**: `docs/specs/strategies/live-mg-bb-reversal.md` — **status: rejected**
- **테스트**: `tests/backtest/test_live_mg_bb_reversal.py` (10건 통과)
- **사전등록 가설**: rejected 된 자매 `live-bb-lower-bounce` (PF=0.922) 의 volume-MA 게이트를 **bullish engulfing OR hammer** 캔들 구조 게이트로 교체하면 false bounce 가 통계적으로 유의하게 줄어드는가?
- **결과 (1y, BTC+ETH, 1m+15m, 8 R/R × 2 freq = 16 조합)**: **falsified**. 모든 조합 PF<1 (범위 0.62~0.83), expectancy 음수. 캔들 구조 필터는 통계적 엣지를 만들지 못했다.
- **자매 5종 + momo-btc-v2 5y 결과(`9553e87`)와 일관**: live-scanner 군 전체가 PF<1 NET LOSER 로 확정된 흐름의 연장선.
- **production 등록**: ❌ 영구 lock. 재활성화는 진입 신호 *재설계* 후에만 (stop/TP 튜닝으로 PF<1 못 넘김이 sweep 으로 증명).
- 상세 결과 표: spec 의 §"1y 검증 결과 (2026-05-20)" 참조.
