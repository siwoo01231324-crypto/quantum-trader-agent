# tests/

## 목적
pytest 단위 테스트. 도메인별 서브디렉토리로 그룹화 (`src/` 모듈 구조와 1:1 대응).

## 구조

### 도메인 폴더

- `signals/` — 팩터 라이브러리 (#67, #71)
  - `test_signals.py` — RSI Wilder 계산, divergence, lag-1 룩어헤드 방지
  - `test_signals_registry.py` — `@register` 데코레이터, `FACTOR_REGISTRY`, `compute()` input-filtering
  - `test_factor_sma.py` · `test_factor_atr.py` · `test_factor_macd.py` · `test_factor_bollinger.py` · `test_factor_realized_vol.py` — 신규 5팩터 수치 검증 + pandas-ta cross-check (skipif)
  - `test_factor_cache.py` — FACTOR_SCHEMA 장형식 + Parquet round-trip + UTC 강제
  - `test_lookahead_guard.py` — append-tail-bar causality 검증 (전 등록 팩터 parametrize)

- `backtest/` — 백테스트 엔진 + 전략 (#67, #71)
  - `test_backtest_engine.py` — 이벤트 루프·포트폴리오·MDD halt·Sharpe/MDD
  - `test_backtest_factor_integration.py` — engine ↔ registry 통합 + 70k-bar O(N²) 퍼프 게이트(`@pytest.mark.slow`)
  - `test_metrics.py` — Prometheus 메트릭 방출 검증 (10종) (#26)
  - `test_momo_btc_v2.py` — momo-btc-v2 전략 프로토콜 준수·진입/청산
  - `test_momo_btc_v2_migration.py` — US-005 마이그레이션 회귀: legacy(compute_rsi 직접) vs new(context["factors"]["rsi"]) bit-identical 검증 (equity_curve + trades 전 필드, 3 sizing_mode × 2 assertion = 6 테스트)
  - `test_run_backtest.py` — CLI 러너·프론트매터·doc_agent·풀 파이프라인

- `data_lake/` — 데이터 레이크 (#20, #67)
  - `test_schema.py` — validation + partition_path
  - `test_fetch_candles.py` — Binance REST → Parquet (responses HTTP mock)

- `obsidian/` — 볼트 MCP + 자동 초안 (#47, #51, #52, #53, #55)
  - `test_obsidian_mcp.py` · `test_obsidian_mcp_sparql.py` — MCP 7 도구 + sparql 우선순위/쓰기 차단
  - `test_doc_agent.py` — 백테스트/인시던트/포스트모템 초안 생성 + 감사 로그
  - `test_migrate_frontmatter.py` — 경로 기반 type 추론·idempotency·H1 추출·티커 ID 보존

- `ontology/` — RDF/SHACL/GraphDB (#47, #54, #55)
  - `test_ontology_sync.py` · `test_ontology_sync_push.py` — 프론트매터 → RDF 동기화, `--push-graphdb` CLEAR+INSERT
  - `test_ontology_roundtrip.py` — Protégé round-trip 의미 보존 (golden fixture)
  - `test_shacl.py` — SHACL 10 규칙별 위반/준수 (`tests/fixtures/shacl/`)
  - `test_graphdb_bootstrap.py` — GraphDB 재시도/생성/idempotency (responses mock)

- `brokers/` — 증권사 API 커넥터 (#68)
- `execution/` — 주문 실행 알고리즘 (#25)
- `integration/` — 네트워크/시크릿 필요 (`pytest -m integration`)
- `fixtures/` — 공통 픽스처 (`doc_agent/`, `obsidian_vault/`, `ontology/`, `shacl/`)

### 루트 (단독 파일)

- `test_risk_dsl.py` — 리스크 DSL 파싱·breach 감지 (#24)
- `test_risk_sizing.py` — 포지션 사이징 순수 함수(Kelly·Fractional Kelly·Vol Targeting·EWMA σ) + momo-btc-v2 `sizing_mode` 통합 (#69)
- `test_portfolio_risk.py` — 포트폴리오 리스크 (LW Σ + CVaR + ENB + 평균 ρ) 수치 검증 (#70)
- `test_portfolio_orchestrator.py` — 전략 수익률 등록·CVaR·ENB·상관 경보 파이프라인 (#70)
- `test_kill_switch.py` — Kill Switch 게이트·트리거·CLI dry-run (#27)
- `test_tax.py` — FIFO·양도세/거래세/배당세·연말 CSV (#28)

> 단독 파일은 추가 파일이 생기면 `tests/{risk,portfolio,ops,tax}/` 서브디렉토리로 승격한다. (#69/#70 의 risk/portfolio 는 2+ 파일이라 다음 chore 에서 폴더화 대상)

## 규칙

- 외부 네트워크/디스크 I/O 금지 (순수 단위 테스트)
- 결정적 시드 사용
- 모든 신규 테이블/필드 추가 시 schema validation 테스트 추가 필수
- 신규 DSL 카테고리 추가 시 breach 감지 테스트 필수
- 신규 팩터 추가 시 `signals/test_factor_{name}.py` + `test_lookahead_guard.py` parametrize 확장 필수
- `@pytest.mark.slow` 로 마킹한 벤치마크는 기본 CI 에서 skip (`pyproject.toml::addopts`)
- 서브디렉토리 이동 시 `Path(__file__)` 기준 `parent` 홉 카운트 주의 — `tests/{domain}/` 는 repo root 로 가려면 `.parent.parent.parent`
