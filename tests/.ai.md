# tests/

## 목적
pytest 단위 테스트.

## 구조
- `test_schema.py` — 데이터 레이크 스키마 validation/partition path 테스트 (#20).
- `test_risk_dsl.py` — 리스크 룰 DSL 파싱(invalid YAML 거부) + breach 감지(per_trade/per_day/leverage/drawdown/sector) (#24).
- `test_execution.py` — 실행 알고리즘 4종(TWAP/VWAP/Market/Limit) + KRX 단일가 핸들러 + 모의 체결 엔진 (#25).
- `test_metrics.py` — Prometheus 메트릭 방출 검증 (10종) (#26).
- `test_kill_switch.py` — Kill Switch 게이트·트리거 3종·CLI dry-run 테스트 (#27).
- `test_tax.py` — FIFO 매칭·양도세/거래세/배당세 계산·연말 CSV 리포터 테스트 (#28).
- `test_ontology_sync.py` — 프론트매터 → RDF 동기화 검증 (#47).
- `test_migrate_frontmatter.py` — 경로 기반 type 추론·idempotency·H1 추출·티커 ID 보존 (#52).
- `test_shacl.py` — SHACL 10 규칙별 위반/준수 픽스처 파라미터라이즈 검증 (`tests/fixtures/shacl/`) (#54).
- `test_graphdb_bootstrap.py` — GraphDB bootstrap 재시도/생성/idempotency 테스트 (`responses` HTTP mock) (#55).
- `test_ontology_sync_push.py` — `ontology_sync.py --push-graphdb` 의 CLEAR+INSERT 단일 SPARQL Update 검증 (#55).
- `test_obsidian_mcp_sparql.py` — MCP sparql() 엔드포인트 우선순위/쓰기 차단/정적 체크 (#55).
- `test_ontology_roundtrip.py` — Protégé round-trip 으로 trading.ttl 의미 보존 검증 (golden fixture `tests/fixtures/ontology/trading_after_protege.ttl`) (#55).

- `test_fetch_candles.py` — Binance 데이터 수집·파티션 Parquet 저장 테스트 (responses HTTP mock) (#67).
- `test_backtest_engine.py` — 백테스트 엔진 이벤트 루프·포트폴리오·MDD halt·Sharpe/MDD 메트릭 (#67).
- `test_signals.py` — RSI Wilder 계산·bullish/bearish divergence 감지·lag-1 룩어헤드 방지 (#67).
- `test_momo_btc_v2.py` — momo-btc-v2 전략 프로토콜 준수·진입/청산 시그널·엔진 통합 (#67).
- `test_run_backtest.py` — CLI 러너·프론트매터 업데이트·doc_agent 초안 생성·전체 파이프라인 (#67).
- `test_risk_sizing.py` — 포지션 사이징 순수 함수(Kelly·Fractional Kelly·Vol Targeting·EWMA σ) 수식 레퍼런스·경계값·입력 검증·pandas/numpy 호환·결정성, momo-btc-v2 `sizing_mode` 통합 테스트 (#69).
